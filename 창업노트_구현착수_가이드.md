# 창업노트 — 구현 착수 가이드

작성일: 2026-06-24 · 설계·검증 완료 후 실제 빌드 시작점.

## 1. 레포 전략 — 3 repo + 계약 seam (상세: `창업노트_레포전략_배포토폴로지.md`)
| repo | 내용 | 배포 |
|---|---|---|
| **cunote-web** | Next.js: 웹+어드민+앱API+BFF, DB 마이그레이션, OpenAPI/계약 | Vercel(단일) |
| **cunote-app** | Flutter iOS/Android (앱API 클라이언트 코드생성) | App Store/Play |
| **cunote-ingestion** | Python 수집·변환·추출·정규화 워커 | GCP Cloud Run |

**cunote-web 내부**(pnpm workspace, 분리 가능하게):
```
apps/web/  app/(web)/ · app/(admin)/ · app/api/app/v1/ · app/api/admin/ · app/api/web/
packages/core/        # 비즈니스 로직(Next 비의존) ← 분리의 핵심
packages/contracts/   # OpenAPI · 타입 · grant_criteria JSON Schema
db/                   # 마이그레이션·RLS (단일 소유)
```
**cunote-ingestion 내부**: `collectors/ convert/ extract/ normalize/ jobs/` (DB 스키마 + criteria JSON Schema 소비).
> 계약은 cunote-web 소유. 앱은 OpenAPI→Dart 코드생성, Python은 criteria JSON Schema 소비. 마이그레이션 이중 금지(web 단일 소유).

## 2. API 표면 (초안) — 웹/앱/어드민 네임스페이스 분리(분리 가능 설계)
**앱 API (버전드·토큰 인증)** `/api/app/v1/*` — Flutter + 웹 공용 비즈니스 엔드포인트:
| 메서드 | 경로 | 용도 |
|---|---|---|
| POST | `/api/app/v1/auth/login`·`/auth/{provider}`·`/auth/refresh`·`/auth/logout` | 앱 토큰 발급·회전·폐기(NextAuth 외 토큰 레이어, `app_refresh_tokens` 해시 저장) |
| POST | `/api/app/v1/verify` | 소유권 검증(번호+대표자명+개업일 → 국세청 진위확인) |
| POST | `/api/app/v1/enrich` | 팝빌 기업정보조회 → `company_enrichment_cache` 저장 → `company_profiles` 투영 |
| GET | `/api/app/v1/matches?companyId` | 매칭(적격/조건부/부적격 + rule_trace + 점수) |
| GET | `/api/app/v1/next-question` | 정보이득 1위 progressive 질문 |
| POST | `/api/app/v1/profile/field` | progressive 입력 → 재매칭 |
| GET | `/api/app/v1/grants/:id` · `/roadmap` | 공고 상세 · 2D 로드맵 |

**웹 인증** `/api/auth/*` NextAuth(세션). **어드민** `/api/admin/*` labeling·golden·eval(role gate).
> 비즈니스 로직은 `packages/core`에, 라우트는 얇은 어댑터 → 미래 앱API/어드민 분리가 배포 변경으로 끝남.

## 4. Phase 1 태스크 분해
**T0 기반** — 레포·CI·Supabase·마이그레이션(통합 DB 스키마: `app_refresh_tokens`, `company_enrichment_cache` 포함)·env·RLS 골격.
**T1 수집(K-Startup 우선)** — 신엔드포인트 어댑터 → grant_raw → 필드파서(지역·업력·연령·분야·대상·일정) → grants/grant_criteria. *LLM 없이 척추 매칭 가능*(S0-A/S0_KStartup PoC 실증).
**T2 수집(기업마당)** — RSS + 첨부 hwp5html 변환 → Claude tool-use 추출 → 정규화. dedup.
**T3 보강** — 국세청 진위확인(검증) + 팝빌 `checkBizInfo` 기업정보조회 + 필드캐시 + canonical 투영 + 자가신고 폴백.
**T4 매칭** — 규칙엔진(필수/우대/배제) + 적합도 스코어 + rule_trace + 정보이득. (PoC `poc/kstartup_match_demo.py` 기반)
**T5 서빙** — Next.js 플로우(번호→검증→매칭→progressive→로드맵) + NextAuth.
**T6 계정·권한·개인정보** — companies/user_company/RLS/인가 + 동의 원장 + 앱 refresh 토큰 회전/폐기.
**T7 플라이휠** — 로깅·버전태깅·라벨링 큐·평가 하니스 CI(1일차부터).
**T8 예비창업** — 수기 진입(같은 엔진).

> 순서: T0(+RLS·토큰·동의 골격) → T1·T4(척추 매칭 빠른 가치) → T3·T5 → T2(기업마당) → T6 고도화·T7 병행. T7 로깅은 처음부터.
