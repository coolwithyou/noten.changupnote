# 창업노트 — 사용자 플로우 & 로드맵 설계

작성일: 2026-06-19 · 역할: 진입→매칭→로드맵의 사용자 여정과 화면, 피드백 수집점

## 1. 두 진입 경로
| 경로 | 입력 | 보강 | 비고 |
|---|---|---|---|
| **기창업(메인)** | 사업자번호 | 팝빌 자동보강 | "번호 하나로" 헤드라인 |
| **예비창업(서브)** | 예정 업종·지역·대표자속성 수기 | 없음(자가신고) | 얇은 경로, 같은 엔진 |

## 2. 메인 플로우 (기창업)
```
① 번호 입력
② 소유권 검증  ← 사용자가 [번호+대표자명+개업일] 입력 → **국세청 진위확인 API 3요소 일치**(자기참조 없음). 민감단계는 본인인증 격상. (상세: 계정권한_개인정보 설계)
③ 팝빌 1차 보강 → company_profiles (공개정보 범위). 부족분은 자가신고 폴백
④ 1차 매칭(티저)  → "적격 N건 · 조건부 M건 · 추정 지원금 ₩X"
     · 무거운 정밀조회/저장은 로그인 게이트 뒤
⑤ Progressive 입력  ← 정보이득 1위 질문 1개씩 ("매출 입력 시 3건을 확정/제외 판단")
⑥ 상세 매칭  → 적합도 헤드라인 + 보조배지 + rule_trace 설명 + 신뢰도
⑦ 스코어  → 적합도(확정) / 선정가능성(추정·v1+) / 가치
⑧ 2D 로드맵  → 일정·전략 (아래)
```

## 3. 예비창업 플로우
번호 없이 ③ 대신 **예정 정보 수기입력** → 같은 ⑤~⑧. 프로필 confidence 낮음 → "확인 필요" 비중↑.

## 4. 매칭 결과 화면 (신뢰 계약)
- **헤드라인 = 적합도**, 선정가능성·가치는 보조 배지("추정" 라벨 명시).
- `eligible / 조건부 / 부적격` 구분 표기. 부적격도 **사유와 함께** 노출(숨기지 않음, recall).
- **rule_trace 설명**: "업력 7년 이내 ✅ 4년 2개월 / 비수도권 우대 ➖ 서울 / 운영사 추천 ⚠ 원문확인".
- 정렬·필터: 적합도·가치·마감임박·분야·지역.
- 피드백 버튼: **적합/부적합**(→플라이휠).

## 5. 2D 로드맵 (시간 × 조건) — 핵심 차별화
단순 마감 캘린더가 아니라 **전략 맵**:
| 구간 | 정의 | 예 |
|---|---|---|
| **지금 적격** | 모든 필수 pass | 즉시 신청 대상, 마감일순 |
| **곧 적격** | 시간 충족 시(업력 등) | "3개월 뒤 업력 3년 → 청창사 가능" |
| **준비하면 적격** | 조건 충족 시(인증·서류·고용) | "벤처확인 취득 시 +5건" |
| **조건부(확인 필요)** | unknown 필드 존재 | "매출 입력 시 확정" |

- unknown·미충족 criteria가 곧 **미래 노드**가 됨(스코어 엔진의 정보이득과 동일 소스).
- 타임라인: 마감일 + 미래 적격 도래일 축.

## 6. 알림
- 마감 임박, 신규 적격 공고, **곧 적격 도래**(업력 충족 등), 조건부 해소 가능 안내.
- 채널: 이메일/카카오(NextAuth 연계 계정).

## 7. 신청 연결 (정직한 한계)
- 실제 접수는 각 기관 포털 → **딥링크 + 서류 준비 도움 + 일정 알림**. 대리신청 ❌.
- 서류 초안(향후 LUMEN 연계)·체크리스트 제공.

## 8. 피드백 수집점 (→ 플라이휠)
- 결과 카드: 적합/부적합
- 신청 버튼 클릭(암묵), 저장
- **신청 결과 보고**(선정/탈락) ← outcome, 최고 정답(유인 설계 필요: 알림·인센티브)

## 9. 화면 목록 (개념)
랜딩/번호입력 · 소유권검증 · 로그인(이메일·Google·Kakao) · 매칭 대시보드 · 공고 상세(rule_trace) · 프로필 입력(progressive) · 2D 로드맵 · 알림설정 · (어드민: 라벨링·골든·평가)

## 10. 인증 접점 (NextAuth)
- 1차 매칭 티저는 비로그인 허용, **저장·정밀조회·알림은 로그인**.
- 이메일/PW·Google·Kakao. 세션·계정 Postgres 어댑터.
