기술 · 01 / 08
문제 — 흩어진 공고, 비정형 자격요건
정부지원사업은 채널·형식·조건이 모두 제각각이라, 사람이 한 건씩 읽지 않으면 "우리 회사가 되는지"를 알 수 없다.
1채널 파편화
K-Startup·기업마당 등 수십 개 채널에 분산. 어디에 무엇이 열려 있는지조차 파악이 어렵다.
2비정형 자격요건
자격요건이 본문·HWP 표·첨부 산문에 흩어져 있어, 판정하려면 사람이 일일이 해석해야 한다.
3조건의 미묘함
"업력 7년 이내", "수도권 제외", "신산업은 10년 완화"… 자동화가 어렵고, 잘못하면 거짓 탈락이 난다.
4접수처 파편화 (실측)
K-Startup n=1,000 실측 — 온라인 접수처가 225종(forms.gle 138종 등)으로 갈라져 통합 자동제출 자체가 불가능.
핵심 난제 = 비정형 공고 → 신뢰할 수 있는 구조화 + 설명 가능한 판정. 규모로, 거짓 확신 없이, 데이터가 쌓일수록 정교해지게 푸는 것이 우리의 기술이다.

해법 — 표준 계약 + 설명 가능한 매칭 엔진
모든 공고를 회사와 같은 좌표계(14축)로 환원한 뒤, 결정론 규칙엔진이 적격·근거·다음 행동을 자동 산출한다.
INPUT
비정형 공고
본문 · HWP 표 · 첨부
AI 추출
→
CONTRACT
14축 표준 계약
grant_criteria · confidence · source_span
규칙 판정
→
VERDICT
적격 · 조건부 · 부적격
+ 근거(rule_trace) + 다음 행동
판정 = 단 3줄의 결정론 규칙
1. 필수·배제 중 하나라도 fail → 부적격
2. fail은 없는데 unknown → 조건부
3. 전부 pass → 적격
"AI는 읽고,
규칙이 판정한다."
블랙박스가 당락을 정하지 않는다. 판정은 늘 재현·감사 가능.

기술 03 / 08
기술 · 04 / 08 · 차별점 ①
AI는 읽고, 규칙이 판정한다
추출(AI)과 판정(규칙)을 구조적으로 분리해, LLM이 틀려도 그 영향이 "입력 품질"에 갇히게 만든다.
추출 = AI (Claude)
tool-use 구조화 출력 강제 — 자유 텍스트 JSON 금지
필드별 confidence + source_span(원문 근거) 동봉
controlled vocabulary만 허용 (업종·지역·인증)
근거 없으면 null — 날조 금지
판정 = 결정론 규칙엔진
조건을 한 줄씩 비교 → pass · unknown · fail
3줄 규칙으로 적격·조건부·부적격 확정
재현·감사 가능 · 버전드(RULESET_VERSION)
결측 = unknown → 거짓 탈락 금지
예: "업력 7년 이내(필수) — 귀사 4년 2개월 ✅" · "운영사 추천 필요(필수·자동판정 불가) — ⚠ 원문 확인". 설명이 곧 신뢰의 핵심이자 라벨링 단위.

살아있는 월드 + 지속개선 플라이휠
시간이 흐르고 데이터가 쌓일수록 정교해지되, 적격 판정은 언제나 결정론·설명 가능하게 유지한다.
살아있는 월드 — 시간 기반 재매칭
eligible_from / eligible_until 전이 날짜를 사전계산
가벼운 일일 잡이 그날 전이건만 flip + 알림
프로필·공고 변경은 영향받는 (회사×공고)만 재매칭
→ 매일 전수 재평가(N×M) 금지 = 비용 가드
플라이휠 — 1일차부터 점수·버전·라벨링
모든 산출물에 model·prompt·ruleset·scoring 버전 태그
저신뢰·피드백·outcome → 라벨링 큐 → 골든셋
CI 평가 게이트 — 회귀 시 활성화 차단
backfill은 R2 원본으로 재추출(재크롤 0)
① 추출 정확도
② 매핑 자산
③ 스코어 가중치 (learn-to-rank)
④ 선정가능성 모델
