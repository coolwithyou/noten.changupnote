# 창업노트 문서 허브

이 폴더는 창업노트 시스템을 **읽는 사람에 맞춰** 정리한 문서 모음입니다.
루트에 흩어져 있는 `창업노트_*.md`는 "설계자/개발자용 설계 문서"이고,
이 `docs/` 폴더는 **읽는 대상별로 카테고리를 나눈 정돈된 문서**를 둡니다.

## 카테고리

| 폴더 | 대상 독자 | 무엇을 담나 |
|---|---|---|
| [`explainers/`](./explainers/) | **비개발자 · 신규 합류자 · 사업/기획** | 시스템이 "무엇을, 왜, 어떻게" 동작하는지 코드 없이 풀어쓴 해설 |
| `architecture/` _(예정)_ | 개발자 | 기술 설계·스키마·인터페이스 상세 (현재는 루트 `창업노트_*.md`가 대신함) |
| `guides/` _(예정)_ | 운영자 · 개발자 | 셋업·배포·데이터 파이프라인 운영 방법 |
| `decisions/` _(예정)_ | 전원 | 주요 의사결정 기록(ADR) — "왜 이 방식을 택했나" |

> 새 카테고리가 필요해지면 이 표에 한 줄 추가하고 폴더와 `README.md`를 만든다.

## 문서 작성 규칙

1. **독자를 먼저 정한다.** 비개발자용이면 코드·전문용어를 최소화하고 비유·예시·표로 설명한다.
2. **코드가 정답이다.** 해설 문서는 코드 동작을 풀어쓴 것이므로, 로직이 바뀌면 해당 문서도 갱신한다. 각 문서 하단에 "근거 코드 위치"를 남긴다.
3. **카테고리 README의 목록에 등록한다.** 새 문서를 추가하면 해당 폴더 `README.md`의 표에 한 줄 추가한다.
4. **파일명은 한국어 슬러그**로 읽기 쉽게 (`핵심-매칭-로직-쉽게-이해하기.md`).
