# 창업노트 검수팀 시작 가이드

> 대상: 공고 분석 검수팀
> 검수 화면: https://ops.changupnote.com/review
> 이 문서에는 비밀번호가 없다. 개인별 로그인 정보는 운영 담당자가 별도 문서로 전달한다.

## 1. 로그인

1. 위 주소를 열고 **Google로 계속**을 선택한다.
2. 운영 담당자가 등록한 본인의 `@noten.im` 계정으로 로그인한다.
3. Google 로그인이 불가능할 때만 개인 전달 문서의 임시 비밀번호로 이메일 로그인한다.
4. 로그인 후 `검수 배정 큐`가 보이고 다른 운영 메뉴에는 접근할 수 없는 것이 정상이다.

개인 전달 문서와 임시 비밀번호는 공용 Slack·메일 스레드에 올리거나 다른 사람에게 전달하지 않는다. 로그인 확인 뒤 운영 담당자에게 전달 문서 폐기를 요청한다.

## 2. 이번 검수의 목적

AI가 공고 원문에서 뽑은 자격요건을 실제 매칭에 써도 되는지 확인한다. 문장이 예쁘게 요약됐는지가 아니라 다음 질문에 답하는 일이다.

> **이 항목을 이대로 DB에 넣고 매칭 판정에 썼을 때, 원문과 다른 결론이 나오는 기업이 존재하는가?**

검수 완료를 기다리느라 사용자에게 공고가 늦게 노출되지는 않는다. 따라서 속도를 위해 추측해서 `정확`을 고르지 말고, 원문으로 확정할 수 없으면 `판단 불가`와 근거를 남긴다.

## 3. 기본 작업 순서

1. `검수 배정 큐`에서 `검수 시작`을 누른다.
2. 화면 가운데의 `현재 검수 질문`을 먼저 읽는다. 예: `AI 분석: 이 공고의 기업 규모 조건은 “소기업”이며 필수 조건입니다. 공고 원문에서도 실제로 이렇게 요구하나요?`
3. 왼쪽에 기본으로 열린 `AI 분석 문서`에서 분석 맥락을 확인하고, `저장된 공고 원문`으로 전환해 AI가 근거로 잡은 문구와 실제 요구 조건을 대조한다.
4. 저장된 원문만으로 부족하면 `공고 원문 페이지` 또는 `공고 첨부 문서`의 HWP/HWPX 미리보기를 새 탭으로 연다.
5. 판정을 선택하고 필요한 사유를 적은 뒤 `저장하고 다음 미판정 항목`을 누른다.
6. 저장이 끝나면 다음 미판정 필드가 자동으로 열린다. 오른쪽 `전체 필드`에서 저장 완료·저장 전 변경·미판정 상태를 확인한다.
7. 마지막 항목에서는 `저장하고 공고 검수 완료`를 누른다. 오른쪽에 완료 안내가 나타나면 `공고 목록으로 돌아가기`를 눌러 다음 공고를 연다.

체크 아이콘과 저장한 판정명이 표시된 필드는 이미 저장된 항목이므로 다시 판단할 필요가 없다. 수정할 때만 해당 필드를 눌러 판정이나 사유를 바꾼 뒤 다시 저장한다. 한 공고를 한 번에 끝내기 어려워도 필드별 저장 내용은 유지된다. 저장 충돌 알림이 나오면 새로고침 후 최신 내용을 확인하고 다시 입력한다.

배정 큐의 `완료` 탭에서는 이미 끝낸 공고만 모아 볼 수 있다. 판정을 다시 확인하거나 수정해야 한다면 `결과 보기·수정`으로 공고를 열고, 필요한 필드만 변경해 저장한다.

### 화면 배치

- **왼쪽 — 판정 근거:** 저장된 공고 원문, AI 분석 문서, 공고 원문 페이지, HWP/HWPX 첨부 미리보기
- **가운데 — 현재 검수 질문:** AI가 어떤 값을 어떤 조건으로 판단했는지와 원문 인용, 판정·사유, 저장 버튼
- **오른쪽 — 전체 필드:** 공고에 배정된 모든 필드와 각 필드의 저장 상태·판정 결과

원본 JSON은 기본 검수에 필요하지 않다. 개발 담당자와 기술 정보를 확인해야 할 때만 현재 질문 아래의 `기술 정보 보기`를 펼친다.

## 4. criterion 판정 기준

| 판정 | 언제 선택하나 | 사유 |
|---|---|---|
| **정확** | 표현이 축약돼도 원문과 매칭 결론이 달라지지 않는다. | 선택 |
| **수정 필요** | 요건은 원문에 있지만 값·연산자·필수/우대/결격 종류·축이 잘못돼 결론이 달라질 수 있다. | 필수 |
| **오류** | 요건 자체가 원문에 없거나 다른 문구를 자격요건으로 오독했다. | 필수 |
| **판단 불가** | 제공된 원문·분석 문서만으로 확정할 수 없다. | 필수 |

`수정 필요` 사유에는 가능하면 다음 두 가지를 함께 쓴다.

- 무엇이 틀렸는지
- 원문 기준으로 어떤 값이나 구조가 맞는지

예: `39세 이하만 근거가 있고 19세 하한 근거는 확인되지 않음. 하한을 제거해야 함.`

## 5. 빈 축 판정 기준

빈 축 항목은 해당 주제의 조건을 AI가 놓쳤는지 확인하는 항목이다.

| 판정 | 언제 선택하나 |
|---|---|
| **없음 확인** | 원문 전체에 그 축의 필수·우대·결격 조건이 없다. |
| **누락 있음** | 원문에 그 축의 조건이 있는데 분석 결과가 잡지 못했다. |

`누락 있음`에는 누락된 원문 문구와 필수·우대·결격 중 어디에 해당하는지 적는다.

## 6. 항목당 확인 루틴

1. **근거 실재 여부** — 인용 문구가 공고 원문에 실제로 있는지 확인한다.
2. **값과 연산자** — 수치·기간·목록·플래그와 `이상/이하/초과/미만` 방향을 대조한다.
3. **종류** — 필수·우대·결격이 원문 취지와 맞는지 본다.
4. **축** — 업력·지역·업종 등 기존 축 대신 `other`로 잘못 분류되지 않았는지 본다.
5. **정보 손실** — 원문 목록 일부가 빠지거나 수치 조건이 단순 문장으로 내려가 기계 판정이 불가능해지지 않았는지 본다.

## 7. 자주 흔들리는 경계

- `서류 미비 시 배제`, `허위 기재 시 제재`, `개별 공고문 참고` 같은 절차·제재·위임 문구를 자격요건으로 만든 경우: 문구는 실재하므로 대개 `오류`보다 `수정 필요`가 맞다.
- 원문에는 수치 조건이 있는데 `text_only` 문장으로만 만든 경우: 기계 판정이 불가능해지므로 `수정 필요`다.
- 같은 취지의 항목이 중복된 경우: 하나만 `정확`, 나머지는 중복 사유를 적고 `수정 필요`로 둔다.
- 열거형 목록에서 원문 항목 일부가 빠진 경우: 빠진 기업의 판정이 달라지므로 `수정 필요`다.
- 원문과 분석 문서만으로 정말 확정할 수 없는 경우: 추정하지 말고 `판단 불가`로 둔다.

## 8. blind 표본

`독립 표본` 배지가 있는 항목은 두 검수자의 독립 판정을 비교하는 표본이다. AI 판정과 다른 검수자의 답이 보이지 않는 것이 정상이며, 원문만으로 판단한다. 서로 답을 맞추지 않는다.

두 판정이 다르면 운영 담당자가 별도 3심으로 확정한다. 검수자가 직접 상대 판정을 확인하거나 조정할 필요는 없다.

## 9. 첫 공고 온보딩

첫 번째 공고는 저장 전 운영 담당자 또는 창업자 판정과 대조한다.

1. 먼저 각자 독립 판정한다.
2. 판정과 사유를 비교한다.
3. 기준이 달랐던 항목은 이 문서의 어느 규칙을 적용해야 하는지 합의한다.
4. 합의 뒤 첫 공고를 저장하고 나머지 배정을 진행한다.

## 10. 문제가 생겼을 때

운영 담당자에게 아래 정보만 전달한다.

- 발생 시각
- 보고 있던 공고 제목 또는 화면 주소
- 오류 메시지
- 새로고침 후에도 같은지 여부

원문·개인 전달 문서·임시 비밀번호를 공용 채널에 붙여 넣지 않는다. 배정 큐가 비어 있으면 임의로 다른 운영 메뉴를 찾지 말고 배정 여부를 문의한다.

## 11. HWP/HWPX 첨부 미리보기

1. 왼쪽 `공고 첨부 문서`에서 파일명을 누르면 RHWP Studio 미리보기가 새 탭으로 열린다.
2. `HWPX 비표준 감지` 안내가 뜨면 `자동 보정 (권장)`을 누른다.
3. 로컬 글꼴 안내가 뜨면 별도 권한 없이 확인하려는 경우 `대체 글꼴로 보기`를 누른다.
4. 상단에 `N페이지를 불러왔습니다`가 표시되면 페이지 수와 문서 내용을 확인한 뒤 원래 검수 탭으로 돌아와 판정한다.
5. 필요하면 `원본 다운로드`로 출처 파일을 내려받을 수 있다.
6. 미리보기 화면에서 문서 내용이 바뀌어 보여도 변경 사항은 저장되거나 공고에 반영되지 않는다.

미리보기가 열리지 않으면 공고 원문 페이지의 첨부 링크로 먼저 확인한다. 그래도 확인할 수 없으면 추측하지 말고 `판단 불가`를 선택하고 오류 메시지와 파일명을 운영 담당자에게 전달한다.
