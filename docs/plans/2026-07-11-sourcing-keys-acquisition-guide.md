## 0. 공통 — 키를 넣고 확인하는 법

1. `apps/web/.env.local` 파일을 연다(없으면 생성).
2. 각 항목의 `변수명=값` 을 추가한다.
3. dev 서버를 **재시작**한다: `pnpm -F web dev` (`.env.local`은 기동 시 로드).
4. `http://localhost:3000/dev/service-data` 에서 사업자번호 조회.
5. 필드 상태 배지: **키 없음** → **키 있음·커넥터 배선 대기**(키 넣은 직후) → **live**(Phase 2 배선 후 정상) → **실패**(키 있으나 에러/빈값/스키마 불일치).

## 1. [최우선·무료] data.go.kr(공공데이터포털) OpenAPI

**data.go.kr 실검증 완료 — 세 데이터셋 다 실재.** 계정 인증키 1개로 여러 데이터셋을 쓴다. 이미 국세청(NTS)·공공구매(SMPP)를 쓰고 있으니 **계정과 인증키는 이미 보유 중일 가능성**이 큽니다. 아래 데이터셋만 활용신청하면 됩니다.

### 발급 단계
1. https://www.data.go.kr 로그인.
2. 데이터셋 번호로 검색 → **[활용신청]** → 활용목적 입력 → 신청(대부분 자동승인, 즉시~수시간).
3. **마이페이지 → 오픈API → 활용신청 현황**에서 **일반 인증키** 확인. **기존 `CUNOTE_NTS_SERVICE_KEY` 값(같은 문자열)을 그대로 재사용**하면 형식 문제 없음.

### 신청할 데이터셋 (검증된 URL)
| 데이터셋 | ID | URL | 조회키 | 반환 | 점등 필드 |
|---|---|---|---|---|---|
| 근로복지공단_고용/산재보험 현황정보 | `15059256` | data.go.kr/data/15059256/**openapi.do** | **사업자번호**(`v_saeopjaDrno`) ✓ | 상시인원·성립일·업종 | **employees, insured_workforce 성립** |
| 금융위원회_기업 재무정보 | `15043459` | data.go.kr/data/15043459/**openapi.do** | **법인등록번호**(`crno`) ⚠️ | 매출·부채·자본·자본금·부채비율 | **revenue·financial_health(법인)** ✅실측 live |

> ✅ **금융위 기업재무(15043459)는 실측 live 확인** — 삼성전자 등 실데이터·자본금까지 반환. 단 **법인등록번호로 조회**라 사업자번호→법인번호 브리지 필요(현재 apick만 제공, 팝빌 없음 → 브리지 소스가 별도 과제).
> ❌ **개인사업자재무(15108171)는 실측 결과 익명 집계 통계셋** — 사업자번호로 개별 조회 불가(값이 사업자와 무관). 활용신청은 하셨어도 우리 용도엔 못 씀. **개인 매출·재무는 CODEF 경로**로 채웁니다.

### `.env.local`
data.go.kr는 **개인 인증키 1개가 모든 데이터셋에 통용**되므로, 아래처럼 **공유 키 변수 하나만** 넣으면 신규 data.go.kr 소스 전부가 그 키를 씁니다(소스별 변수는 개별 오버라이드용, 선택):
```dotenv
CUNOTE_DATA_GO_KR_SERVICE_KEY=<data.go.kr 개인 인증키>   # 신규 data.go.kr 커넥터 전부가 이 키로 조회
# (선택) 소스별 오버라이드 — 없으면 위 공유 키로 폴백
# CUNOTE_KCOMWEL_SERVICE_KEY=       # 15059256 고용·산재
# CUNOTE_FSC_FINANCE_SERVICE_KEY=   # 15043459 법인재무 (15108171 개인재무는 익명집계셋 → 미사용)
```
> 이미 `CUNOTE_KCOMWEL_SERVICE_KEY`·`CUNOTE_FSC_FINANCE_SERVICE_KEY`를 넣으셨다면 그대로 동작합니다(커넥터가 소스별 변수 → 공유 키 순으로 해석). **국세청(business_status)은 `CUNOTE_NTS_SERVICE_KEY`를 별도로 읽으니**, 같은 인증키 값을 이 변수에도 넣어두면 국세청 휴폐업 최신조회가 켜집니다.

## 1-b. [무료·파일 다운로드] 중대재해 명단 (키 불필요 · Phase 2 배치)

`15090150` 고용노동부_중대재해 발생 사업장은 **파일데이터(CSV)**입니다 — URL은 `data.go.kr/data/15090150/`**`fileData.do`** (검색 시 "파일데이터"로 필터). **런타임 API 키가 아니라 파일을 내려받아 배치 적재**하는 방식이고, **사업장명·소재지만 있고 사업자번호가 없어** 상호 매칭이 필요합니다. → 키 확보 대상이 아니라 Phase 2 배치 파이프라인 사안. (같은 계열: `15090011` 중대산업사고, `15090000` 사망 2명 이상 등도 파일·상호 매칭.)

## 2. [보유분 활용·무료] CODEF 데모 (개인 매출·결격)

개인사업자 매출·결격의 **국세청 확정·실시간** 경로. 데모 일 100건 무료. 이미 데모키 보유 시 상품 신청만 확인.
> 참고: 위 15108171(개인사업자재무)이 무동의 부분 커버를 주므로, CODEF는 **전수·확정 보완**(연 1회·성실신고 대상 한계를 메움).

### 발급 단계
1. https://codef.io 회원가입/로그인.
2. **데모버전 신청**(일 100건 무료).
3. **상품 신청**: `사업자등록증명`, `부가세과세표준증명` (이후 확장: 납세증명·표준재무제표·4대보험 명부).
4. **키관리**에서 `clientId` / `clientSecret` / `publicKey` 확인.

### `.env.local`
```dotenv
CODEF_CLIENT_ID=<clientId>
CODEF_CLIENT_SECRET=<clientSecret>
CODEF_PUBLIC_KEY=<publicKey>
CODEF_ENVIRONMENT=demo
```
> 실측용 실계정 3종(법인 1 · 일반과세 개인 1 · **간이/면세 개인 1**)과 대표자 본인 휴대폰 필요. 샌드박스는 추가인증 미지원.

### 점등 필드
**revenue(개인 확정), founder_age(간편인증 입력), tax_compliance(개인), insured_workforce 피보험자수(정밀)**

## 3. [유료·계약] NICE BizAPI (법인 결격·재무)

법인의 체납·신용불량·법정관리는 NICE만 사업자번호로 조회(법인 신용정보 동의 불요). **데모 테스트앱은 무료지만 고정 샘플 응답** → 구조·파싱까지만 검증, 실데이터는 계약.

### 발급 단계
1. https://www.nicebizline.com 회원가입/로그인.
2. **API 신청 → 테스트앱**(고정응답, 무료)으로 구조 검증.
3. 실데이터는 **계약(견적)** — 법인 결격 3콜(OCCD01/03/06)+재무(OCOV06) 단가 문의.
4. `client-id` / `client-secret` 확인.

### `.env.local` (실제 삽입된 변수명 기준)
```dotenv
NICE_BIZ_CLIENT_APP_KEY=<App Key>      # NICE 콘솔의 App Key → 인증 헤더로 매핑
NICE_BIZ_CLIENT_SECRET=<Secret Key>
NICE_BIZ_ENVIRONMENT=demo              # 계약 후 production
```
> 실제 인증 헤더명(`client-id`/`client-secret` 등)은 NICE BizLine OpenGate 문서로 커넥터 구현 시 확인.
### 점등 필드
**tax_compliance(법인 체납), credit_status(법인 채무불이행·부도·법정관리·파산), 자본금**
