# 공모 딥분석 층화 확대 실험 (30→100건) 실행 계획

> 🔴 **미착수** (2026-07-21 수립). 착수 전제 3가지: ① 파일럿 판정 🟢 5/5 확정
> (`docs/research/2026-07-21-공모딥분석-검수집계-판정.md` — 완료. 단 식품 공고
> export_performance 빈 축 1건 추가 판정·재집계 필요) ② 선행 구현 5건(§5) ③ CALIBRATION
> 외부 대조(§6). 설계는 3관점 멀티에이전트(통계·운영·구현) 산출을 종합한 것.

## 1. 목표

파일럿(3건)의 🟢 판정을 **운영 투입 판단이 가능한 표본 크기(30→100건)에서 확증**한다.
파일럿의 한계 — 소표본(criterion 28건), 본문 두께 우선 선정의 낙관 편향, 단일 검수자
확인 편향 미측정 — 를 층화 표본·사전 등록된 판정 규칙·편향 완화 절차로 해소한다.

## 2. 게이트 — 6종 (구조화 게이트 승격)

기존 5종(`ANALYSIS_LAB_GATES`, contract.ts) + **신규: 정확 확정 B 중 구조화(≠text_only)
비율 ≥ 50%** (승격 결정과 근거는 판정 문서 §6). A→B 배수는 관찰 지표로 유지.

**판정은 점추정이 아니라 신뢰한계로 한다**: 정밀도·구조화는 95% Wilson **하한**이 기준
이상, 오류율·누락은 95% **상한**이 기준 이하일 때 통과. criterion이 공고 내 군집이라
iid가 아니므로 공고 단위 클러스터 부트스트랩 CI를 병기한다(ICC는 30건 집계에서 첫 실측).

## 3. 표본 설계

### 층화·쿼터 (cohort.ts가 이미 산출하는 신호만 사용)

- **기본 층화 6셀** = 소스(kstartup/bizinfo) × 본문 두께 3티어(두꺼움 >30KB / 보통 2~30KB /
  얇음 <2KB·본문성 없음 — rankByBodyMarkdown 티어 재구간화)
- **쿼터(셀 아님, 최소 보장)**: 통합공고(제목 정규식 판별, 신설 필요) ≥4건 · 현행 A≥3
  공고 ≥6건(커버리지 게이트 분모 방어) · 혜택 7 family 각 ≥2건(다중 라벨이라 층화 부적합)
- 전체 교차는 셀 폭발로 기각 — 2×3=6셀이 n=30에서 셀당 5건을 보장하는 상한

### 층별 재고 실측 (2026-07-22, read-only 카운트 — 설계 수정 반영)

| 소스 | 두께 | open | (그중 마감 4주+) | A≥3 | A=0 | 통합공고 |
|---|---|---|---|---|---|---|
| kstartup | 두꺼움(>30KB) | **2** | 1 | 0 | 0 | 1 |
| kstartup | 보통(2~30KB) | **4** | 1 | 0 | 0 | 0 |
| kstartup | 얇음/없음 | 340 | 108 | 214 | 0 | 1 |
| bizinfo | 두꺼움(>30KB) | 48 | **0** | 22 | 25 | 1 |
| bizinfo | 보통(2~30KB) | 278 | 2 | 195 | 66 | 4 |
| bizinfo | 얇음/없음 | 833 | 184 | 641 | 1 | 10 |

실측이 깨뜨린 전제 2개와 수정:

1. **kstartup markdown 재고가 6건뿐** → 소스×두께 균등 6셀 성립 불가. 원인은 첨부 변환
   적체(전사 변환률 14.2%)다. 수정: **1단계 30건은 bizinfo 두꺼움/보통 중심 + kstartup
   markdown 보유 6건 전량 + 얇음 층**으로 구성한다. open 모집단 자체가 bizinfo ~77%라
   대표성 왜곡은 우려보다 작다. **kstartup 층은 변환 서버 재배포 → open 공고 첨부 일괄
   변환 후 2단계(+70건)에서 확보**한다 — 이 변환은 현황평가 §5.2-8이 지목한 제품 최대
   지렛대(recommendable 0.61%의 최대 원인)와 같은 작업이라 실험 전용 비용이 아니다.
2. **applyEnd ≥4주 필터는 bizinfo 두꺼움을 0건으로 만든다** → 필터 폐기. 대신:
   cohort는 grantId 고정이라 **공고가 마감(closed)돼도 행이 남아 있는 한 검수 가능**
   (cohort.ts는 행 생존만 확인). 리스크는 행 삭제뿐이므로 재수집 파이프라인의 삭제 정책
   확인(§7)과 배치 단위 빠른 완주(분석 1일 + 검수 1주 내)로 대체한다.
- A 충실도 쿼터는 재고 충분(A≥3 수백 건). bizinfo 두꺼움에 A=0이 25건 — 커버리지 분모
  구성이 자연스럽게 섞인다. 통합공고 전체 17건 — 쿼터 ≥4 충족 가능(대부분 얇음 층).
- 혜택 family별 재고는 SQL로 못 세므로(코드 파생) 선정 시점에 확인.

### n과 단계 — 30건(진단) → 100건(확증)

파워 계산 근거(스파이크 사전 정보: 공고당 criterion 9.3건, 정밀도 96.4%):

- **30공고(~280 판정)**: 정밀도(참 0.93에서 파워 ≈1.00, ICC 0.2 보정 후에도 0.99)·
  오류율(참 2%에서 ≈1.00 — 파일럿 n=28은 0건이어도 "3의 법칙" 상한 10.7%라 확정 불가했음)·
  누락(참 0.5/공고에서 0.92) 게이트에 충분. n=280이면 관측 정밀도 ≥85.0%가 Wilson 하한
  80% 통과선.
- **100공고**: 신규 구조화 게이트만 참값 하락 시나리오(63%→55~58%)에서 ~900 correct가
  필요 — 이것이 2단계의 통계적 존재 이유. n=930이면 관측 정밀도 ≥82.6%가 통과선.
- 1단계 30건은 6셀 균등 5건 + 쿼터 — **어려운 층(얇은 공고·통합공고) 의도적 과대표집**
  (진단 목적). 2단계 +70건은 모집단(open+markdown 보유) 비례 배분 + 셀 최소 8건 보정,
  최종 게이트는 원시 집계와 사후층화 추정 병기(표본 구성 조작에 의한 통과 차단).

### 30건 중간 판정 규칙 (사전 등록 — 사후 변경 금지)

30건 시점에는 어떤 경우에도 최종 PASS를 선언하지 않는다(진단·확증 역할 분리):

| 판정 | 조건 | 조치 |
|---|---|---|
| **STOP** | 정밀도 점추정 <80% (correct <224/280) / wrong ≥38건(하한>10%) / 총 누락 >30건 | 프롬프트 재설계 후 30건 재실행 |
| **ITERATE** | 정밀도 [80%, 85%) 회색지대 / 누락 (0.67, 1.0]/공고 / 구조화 <50%가 전 층에 걸침 | 확대 보류, 원인 층만 보강 재실행 |
| **GO** | 6게이트 점추정 전부 통과 + 정밀도 Wilson 하한 ≥80% + 판정 ≥10건인 어떤 셀도 정밀도 <60% 없음 | +70건 착수 |

- 구조화 게이트 참값이 50~55% 회색지대면 100건으로도 판정력 부족 — 그 경우 "게이트 실패"가
  아니라 **"판정 불능"으로 기록하고 기준 재협상**(사전 등록해 사후 합리화 차단).
- 100건 최종 집계에는 30건 데이터 포함. **파일럿 3건은 게이트 판정 표본에서 제외한다**
  — 구조화 게이트 수치(50%)를 유도한 데이터가 판정 표본에 재진입하면 순환(circular
  analysis)이다(calibration §4). 집계 코호트 필터의 기본 동작이 이미 제외하므로 구현
  변경은 없고, 파일럿 포함치(`--all`)는 민감도 참고로만 병기한다.
- **동결(사전 등록 실효화)**: 30건 착수 전 프롬프트 v2·검수 가이드(§5 사례집 포함)·판정
  규칙(이 문서 §3)을 동결하고 **동결 시점 커밋 해시를 이 문서에 명시**한다. 수정은
  STOP/ITERATE 판정 후에만 — 중도 변경 시 이전 런과 병합 불가(v1/v2 전례).

### 신규 축 승격 규칙 (사전 등록 — calibration §3 보강)

taxonomy 제안의 축 승격은 30·100건 집계에서 다음 3조건 전부 충족 시에만 검토한다
(외부 정량 표준 부재 — "≥3"은 내부 정의임을 명기):

1. **독립 반복**: 같은 공고 재실행을 제외한 **서로 다른 공고 ≥3건**에서 같은 취지의 제안
2. **리트머스**: 기존 22축·text_only로 담으면 **판정 결과가 달라지는 기업이 존재**
3. **실현성**: evaluator·질문 문항·기업 측 소싱 경로가 구현 가능(축 1개 = evaluator+사전+
   질문+검수 부담의 곱 — 현황평가 §3.1 복잡도 상한 경고)

## 4. 운영 계획

### 검수 부담 (파일럿 실측 기반)

- 파일럿 실측: 3건을 51분(심야 버스트) — 식품 26항목이 9분(항목당 ~21초, 가이드 "30초
  루틴"과 정합). 확대 코호트 예상 24.7항목/공고 × 25~30초 + 고정비 5~8분 = **공고당
  중심 17분(10~35분)**.
- **페이스: 세션 60~90분 × 4~5건, 주 4~5세션 = 주 15~20건 → 30건 약 2주.** 장시간 연속
  검수는 정확/수정 경계 드리프트(피로 시 관대화 → 허위 통과) 위험 — 세션 분할이 곧 품질
  통제다. 매 세션 첫 공고는 가벼운 것으로.
- **100건 전수 검수(5~7주)는 비권장** — 2단계 기본안은 "분석 +70건(기계 지표·비용·구조화율·
  taxonomy 제안은 전수 수집) + **검수는 층별 균등 +40건**(누적 70건 검수)". 단 30건 집계에서
  구조화 게이트가 회색지대(50~58%)면 검수를 전수로 확대(§3의 ~900 correct 요건). 규모
  확정은 30건 집계 후.
- 검수 절감에서 **criterion 샘플링·빈 축 샘플링은 기각** — 재현율 게이트는 빈 축 전수 확인이
  측정 그 자체이고, 커버리지 분자가 correct 절대 건수라 하향 편향이 생기며, aggregate 전면
  재작성을 강제한다. 허용되는 절감은 "공고 간 표집 × 공고 내 전수 검수"뿐.
  spanVerified 자동 정확 처리 등 무검수 승격은 순환성 가드 위반으로 금지.

### 실행 루프 — 10건 배치 × 3 (1단계)

배치마다: **분석 10건(배치 러너) → 검수 10건(세션 2~3회) → `pnpm lab:aggregate`(배치
필터) → 가이드 §5 사례집 갱신 + unsure 재판정**(재판정은 배치 경계에서만 — 배치 중간
규칙 변경은 배치 내 판정 비일관을 만든다. review.json 덮어쓰기 + 최신 dedupe로 안전).

- 분석 선실행 금지(배치 단위) — 프롬프트 개정 시 매몰 손실을 배치당 ~$4로 상한.
- 배치 1 종료 시 조기중단 게이트: wrong률 >10% 또는 공고당 누락 >1건이면 중단·프롬프트 개정.
- 중간 집계(n≈93)는 CI ±6%p로 넓어 스모크로만 — 게이트 판정은 30건 완주 후.
- 착수 전 가이드 §5 사례집에 파일럿 산출 유형("우대 대상 목록의 불완전 열거") 등재 후 **동결**.

### 비용·시간

- 분석: 30건 ≈ **$11.9**, 100건 ≈ **$39.5**(실측 $0.395/공고, $0.17~0.64 입력 크기 연동).
  API 시간 1~2분/공고 — 직렬 100건 3~7시간이라 배치 러너의 동시성 2~3 필수(소량 5건으로
  rate limit 검증 후 본실행).
- 총비용의 실병목은 LLM이 아니라 **검수 노동**(30건 ~8.5시간, 70건 검수 ~20시간)이다.
- **spike-out 백업 루틴**: 검수 파일은 gitignore 대상·디스크 단본 = 골든셋 단본. 주 1회
  아카이브를 운영 절차에 명시.

### 매칭 임팩트 섀도 측정 (30건 완주 시 — 실험의 존재 이유 직접 측정)

추출 정확도(게이트 6종)와 별개로, **"이 criteria가 실제 매칭을 얼마나 개선하는가"**를
30건 검수 확정분으로 실측한다. 검수 확정 criteria를 프로덕션 DB에 쓰지 않고 **섀도로
match_state를 재계산**해(07-11 층화 백필 before/after 방법론 재사용) 다음의 전/후를 잰다:

- 공고당 확정 판정(eligible/ineligible) 조건 수 — **절대량 지표**. 즉시판정률 같은 비율
  지표는 분모(조건 수)가 0.67→9건/공고로 늘어 착시 하락하므로 쓰지 않는다.
- recommendable·eligible 전환 공고 수 (기존 실사용자·팝빌 캐시 사업자 기준)
- "질문 1개로 확정 가능" 공고 수 — 구조화 criteria만 질문 플래너의 재료가 되므로,
  딥분석의 구조화 증분이 해소 루프(실측 해소율 0.74)를 얼마나 확장하는지의 직접 측정
- 신규 exclusion(결격·prior_award)이 제거하는 오추천 수 (정밀도 개선분)

이 측정이 긍정이면 golden_set 승격 트랙(DB 쓰기)의 착수 근거가 되고, 부정이면(공고 측
증분이 기업 측 데이터 부재로 전부 conditional 적체) 소싱 트랙 우선순위를 재조정한다.

   1건뿐이라 경계 재조정 원재료가 없었다 — 판정 문서 §4).
4. **적대적 LLM 재확인** (calibration §4에서 리뷰어 베타 → 확대 실험로 앞당김, 조건부):
   별도 LLM이 correct 판정에 이의 플래그만 제기. **3조건 필수** — ① 확정 권한 없음(항상
   사람, Gate 1 순환성 가드) ② **사후 노출**(사람 판정이 저장된 뒤에만 플래그 표시 —
   앵커링 차단, "판단 먼저 AI 나중" cognitive forcing) ③ 플래그는 재검수 우선순위 신호로만.
   플래그 LLM은 분석 모델(Opus)과 **다른 모델**로(judge 자체 편향 상쇄). 가능하면 외부
   1인과 5공고 중복 검수로 inter-rater κ 확보.

  재협상 경로 사전 등록됨(§3)
