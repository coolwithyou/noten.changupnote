### B. 성능 진단 (구현은 안 함 — 실측 완료)

- DB 자체는 정상: Supabase 서울 풀러, `SELECT 1` 왕복 74ms
- `POST /api/web/teaser` 2.7~5.8초. 범인: `apps/web/src/lib/server/repositories/drizzle.ts`의 `listActiveGrants` **2단계 hydration 쿼리** — `grants × grant_criteria` LEFT JOIN에 `grant_raw.payload`가 붙어 공고당 조건 수만큼 payload 중복 전송. psql 실측 **7,979행 · ~33MB · 3.1초** (활성 공고 2,367건 · criteria 7,868행 · payload 평균 1.8KB). 1단계에서 같은 payload 2,367건(~4.5MB)을 이미 받고 버림
- 캐시 없음: 티저 로드·답변 반영·대시보드마다 위 파이프라인 전체 재실행
- "저장 중…" 무한: `MatchResultsExperience.tsx`의 `saveAndContinue` → `POST /api/web/companies` → `resolveAnonymousProductCompanyProfile` → 캐시 미스 시 팝빌(운영·실과금) 동기 호출(`serviceData.ts`의 `loadPopbillCompanyProfile` 체인). **외부 호출·클라이언트 fetch 모두 타임아웃 없음**. 응답 실패 시엔 로그인 폴백이 있으므로, 고착 = 응답이 아예 안 오는 케이스(외부 API 행 또는 커넥션 풀(최대 4) 고갈)

### C. 환경 이슈 해결 (재발 주의)

- "CSS 전부 깨짐" 원인 = **Cloudflare가 dev.changupnote.com의 dev 자산을 4시간 캐시**(origin `no-cache`를 `max-age=14400`으로 덮어씀 + Turbopack dev 청크 파일명 고정). 메모리 `cloudflare-dev-tunnel-cache-trap` 참조. **영구 해결(Cache Rule) 미적용** — `.env`의 `CLOUDFLARE_TOKEN`은 zone 읽기만 가능, Cache Rules 쓰기 권한 없음

## 남은 작업 (순서대로)

1. **hydration 쿼리에서 payload 분리** — `apps/web/src/lib/server/repositories/drizzle.ts` `listActiveGrants`: 2단계 조인(`rows` 쿼리)에서 `raw: schema.grantRaw` 제거, 1단계 `candidateRows`에서 받은 payload를 Map(source:sourceId→payload)으로 재사용해 `hydrateGrants`에 공급. dedup hydration으로 1단계에 없는 id가 늘어난 경우만 payload를 grant당 1회 추가 조회. 예상 효과: 요청당 ~38MB→~4MB, 티저 3초→1초 미만
2. **공고 유니버스 프로세스 캐시(TTL 1~5분)** — `apps/web/src/lib/server/serviceData.ts` `loadServiceGrantUniverse` 결과(정규화된 배열)를 모듈 레벨 TTL 캐시로. asOf 민감 로직(D-day)은 캐시된 원본에 asOf만 재적용되는지 확인
3. **타임아웃 이중화** — 팝빌 등 외부 호출에 `AbortSignal.timeout(10_000)` 계열, `saveAndContinue`의 fetch에도 타임아웃+실패 시 버튼 복구·토스트
4. **검증**: dev 서버는 사용자에게 기동 요청 후 브라우저로 티저/저장 흐름 실측(이 세션은 Claude in Chrome으로 `performance.now()` fetch 타이밍 측정했음). 회귀: `pnpm --filter @cunote/web test` 해당 스위트 + `activeGrantFilter.test.ts`
5. **production build + 커밋**: dev 서버 내린 상태에서
   ```bash
   pnpm --filter @cunote/web build
   rg -n "<button|<input|<select|<label|<table" apps/web/src/features apps/web/src/app --glob '!**/api/**' | wc -l   # 37 이하
   ```
   커밋은 사용자 승인 후, **`git add -A` 금지**(병렬 세션 변경 혼재) — 이번 세션 변경 파일만 명시 스테이징. 대상: `apps/web/src/app/globals.css`, `apps/web/src/components/ui/{button,input,textarea,select,input-group,badge,dialog}.tsx`, `apps/web/src/components/app/{app-header,notice-card,precision-gauge,verdict-badge}.tsx`, `apps/web/src/features/landing/*`, `apps/web/src/features/match-results/{MatchResultsExperience,TeaserQuestionForm,States,NextQuestionCard,ResultsHero}.tsx` 중 수정분, `apps/web/public/brand/noten-logo.svg`, `docs/design/*`, 이 문서
6. **(사용자 액션 대기) Cloudflare Cache Rule**: 대시보드 → changupnote.com 존 → Caching → Cache Rules → Hostname equals `dev.changupnote.com` → Bypass cache + Browser TTL Bypass. 또는 Cache Rules Edit 권한 토큰을 받으면 API로 생성
