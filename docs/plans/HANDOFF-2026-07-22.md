   pnpm lab:batch -- --limit=2          # rate limit 소량 확인
   pnpm lab:batch -- --limit=10         # 배치 1 본실행 (동시성 기본 2)

- **검수 시트 사고 교훈(07-22 실사고)**: 시트가 빈 채 열리면 기존 검수 로드 실패다 —

  확인할 것. 검수 파일은 디스크 단본(주 1회 백업 루틴 권장).
- **Opus 4.8 파라미터**: temperature/top_p/thinking 전송 시 400 — extractor가 의도적
  미전송. 모델 오버라이드는 env ANALYSIS_LAB_MODEL.
- **dev 서버는 사용자 소유** — 세션이 백그라운드로 띄우지 말 것.
- **파일럿 3건은 게이트 판정 표본에서 제외**(집계 코호트 필터 기본 동작) — `--all`은
  민감도 참고용. 확대 중 프롬프트 v2 동결(수정은 STOP/ITERATE 후에만).
- **코호트 재선정(refresh)**: 검수 보유 공고 보존 가드가 있으나, 확대 실험 중 UI의
  재선정 버튼은 여전히 누르지 말 것(운영 가드).
- batch·aggregate·smoke는 명시적 exit 구현됨(미종료 전례는 verify:service-data 계열).
- **Sonnet 5 A/B 후보는 인트로 가격($2/$10)이 2026-08-31 종료** — 30건 종료 직후 실행하면
  ~$5로 검증 가능(env만 변경, calibration §5).
- 층화 재고는 bizinfo 중심(kstartup markdown 보유 open 6건뿐) — kstartup 층은 변환 서버
  가동 후 2단계에서. grants 행은 인제스트에서 삭제되지 않음(마감은 status UPDATE) —
  코호트 grantId 고정 안전. 단 프로덕션 DB에서 dev 시드/verify 스크립트 수동 실행 금지.
