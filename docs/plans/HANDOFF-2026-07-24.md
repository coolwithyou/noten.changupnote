## 배경 (검증된 사실)

- 해당 검수 건 = kstartup **178352** "해양·수산 [2026 국민 아이디어 제안대회]". LLM 입력(2,646자)에 웹 payload + 포스터.txt만 들어가고 **참가신청서.hwp는 변환 실패로 통누락, 고지도 없었음**.

- **결함 2**: `input.ts`가 markdown 없는 첨부를 **조용히 필터로 제거** → 모델이 `input_missing` 대신 `inspected_no_condition`으로 오판, ops 메타에도 흔적 없음.
- **검수 배분 21개 공고 중 HWP 통누락 4건**: kstartup 178329(3건), 178320(1), 178352(1), bizinfo PBLN_000000000121478(1 — 유일 첨부라 사실상 웹만 분석). 이 4건은 감사 판정 신뢰도에 주석 필요.
- **딥분석 미완 공고 실측** (2026-07-24, 프로덕션 DB): 모집중(open, KST 오늘 포함) 공고 **721건** 중 로컬 분석 런 보유 **10건**, **미분석 711건**. 기간 미상(apply_end null) 958건은 정책상 예외 큐. 분석 완료 목록(35건)은 `spike-out/analysis-lab/<source>__<sourceId>/` 디렉터리명 참조.

- **서브에이전트 3개가 Fable 5 사용량 한도로 두 번 연속 사망**한 이력 있음 — 대량 위임 전 잔여 한도 확인, 큰 작업은 직접 수행이 안전.
- psql 접속: `.env`의 `DATABASE_URL`에 psql이 못 읽는 쿼리 파라미터가 붙어 있음 → `sed 's/?.*$//'`로 제거 후 사용 (읽기 전용 준수).
- SQL 조인 시 `grant_attachment_archives.source`는 enum이라 `a.source::text = n.source` 캐스트 필요.
- `audit_dispatch_notices.input_text`는 발송 시점 스냅샷 — 재분석해도 기존 검수 화면 원문은 안 바뀜(새 런 발행/재배분 필요 여부는 ops 정책 판단).
- 신규 런부터 `[입력 한계 고지]`가 붙어 inputSha256이 달라짐 — 의도된 변화.
- 딥분석 재실행은 실 LLM 비용. Opus 4.8·Fable 5에 temperature 지정 금지(기존 규칙).
