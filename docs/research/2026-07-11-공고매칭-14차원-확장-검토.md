- 작성일: 2026-07-11
- 상태: 검토 완료 → 구현계획 `docs/plans/2026-07-11-matching-dimension-expansion.md`
- 전제: **데이터 소싱 가능 여부를 배제하고, "정확한 매칭을 위해 이론상 필요한 정보"만 기준으로 검토** (사용자 지시)
- 근거 코드: `packages/contracts/src/index.ts:1-16`(14차원), `packages/core/src/matching/match.ts`(판정 엔진), `apps/web/src/lib/server/db/schema.ts:50-65`(pgEnum), `backups/cunote_full_20260627_142340.sql`(실데이터 criteria)

## 1. 결론 요약

**14차원 중 "긍정 요건" 축 13개는 외부 SOTA 대조에서도 타당하다. 그러나 실제 공고에서 가장 빈번한 필수 게이트인 "결격(배제) 조건" 축이 통째로 빠져 있어 전부 `other`(text_only)로 새고 있다.** `other`는 매칭 엔진에서 항상 `unknown` → `needs_core_review`(match.ts:107-109, 24-30)로 처리되므로, 결격 조건은 **차원을 추가하지 않는 한 영구히 자동 판정이 불가능한 구조**다. 파서 개선으로는 해결되지 않는다.

권고: **핵심 5개 차원 신설(14→19) + 선택 3개(→최대 22) + 기존 6개 차원의 값 스키마 확장.**

## 2. 검토 방법

1. **내부 증거**: 코드베이스의 14차원 정의·매칭 엔진·기업 프로필 스키마 전수 확인 + DB 백업의 실제 `grant_criteria`에서 `other`/text_only로 저장된 배제 조건 원문 수집·분해 (6개 공고 → 27개 조건 유형)
2. **외부 대조**: 보조금24·기업마당·K-스타트업(공공 3), 웰로비즈·커넥트웍스·엑시토·혜움·govmatch·혁신의숲(민간 6)의 수집 항목/필터 축 + 대표 사업군(창업·R&D·수출·고용·소상공인·지자체)의 공고 자격조건 실례 리서치

## 3. 현행 체계 진단

### 3.1 구조 요약

- 14차원은 contracts·DB pgEnum·JSON Schema 3중 복제로 계약을 이루며 정의는 일치한다.
- `kind: required | preferred | exclusion`이 이미 있어 **긍정요건/우대/결격의 "성격 축"은 존재**한다. 문제는 결격 조건이 착지할 **차원(dimension)이 없다**는 것.
- `other`는 `CORE_GATE_DIMENSIONS`에 포함되어(match.ts:24-30) unknown 시 `needs_core_review`(scoreDisplay=hidden)로 강등된다. 즉 결격 조건이 붙은 공고는 사실상 추천 불가 상태로 남는다.
- kstartup 파서는 구조화 실패 시 같은 신청대상 원문을 size/industry/certification/other 4개 text_only placeholder에 **복제 배치**한다(normalize.ts:569-688) — 차원이 조건을 못 담을 때 생기는 구조적 누수의 직접 증거.
- bizinfo LLM 추출기의 시스템 프롬프트가 **"세금 체납, 제재, 중복수혜 … 배제 조건은 other text_only exclusion으로 남긴다"고 명시**(llm-criteria.ts:65) — 현행 누수는 우연이 아니라 설계된 임시 조치다.

체납(#1)은 수집 6개 공고 전부에, 신용·제재 계열은 5개에 등장 — 사실상 전 공고 공통 게이트다.

### 3.3 이질 사례 — 개인 대상 공고 (차원 문제 아님)

text_only forced-fit 사례 중 "투자 심사역", "기업 임원·본부장", "2년 이상 재직자", "수요기업(군·방산)" 등은 **기업 매칭 대상이 아닌 교육·포상·네트워킹 공고**다. 차원 신설이 아니라 **공고 단위 audience 분류(기업 대상/개인 대상)를 매칭 파이프라인 진입 전에 두는 상류 게이트**로 풀어야 한다.

## 5. 확장 제안

### 5.1 신설 — 핵심 5차원

| 차원 | 커버 조건 | 사용자에게 받을 정보 |
|---|---|---|
| `tax_compliance` 조세·공과금 체납 | 국세·지방세 체납 배제(전 공고 공통), R&D는 관세·4대보험료까지 | 국세/지방세 체납 여부, 납부유예·징수유예 승인 여부(승인 시 신청 가능 예외 빈출) |
| `credit_status` 신용·금융 상태 | 채무불이행·연체, 부도, 회생·파산·법정관리, 금융질서문란, 압류, 보증사고·보증제한 | 연체/채무불이행 여부, 회생·파산 진행 여부(+변제계획 정상 이행 — 예외), 자산 압류 여부 |
| `sanction` 제재·참여제한 | 참여제한 중, 부정수급·환수, 보조금법 위반, 의무 불이행, 임금체불 명단, 협약 위반 | 참여제한 여부·종료일, 부정수급 이력 (일부 공고는 특수관계 기업 포함) |
| `financial_health` 재무 건전성 | 부채비율 상한, 부분/완전 자본잠식 배제, 이자보상배율 (정책자금·R&D 표준 조항) | 최근 결산 부채총계·자본총계·자본금·**자산총액** (자산총액은 중기법 `size` 판정 정확도까지 개선하는 이중 효과) |
| `insured_workforce` 고용보험·4대보험 | 고용보험 성립 요건, 피보험자 수 기준, 감원 이력 배제 (고용부 사업 필수) | 고용보험 성립 여부, 피보험자 수, 최근 6~12개월 인위적 감원 유무 |

`employees`(상시근로자)와 `insured_workforce`(피보험자)는 산정 근거법이 달라 같은 회사도 숫자가 다르다 — 통합 불가.

### 5.2 신설 — 선택 3차원 (사업군 특화)

| 차원 | 근거 | 받을 정보 |
|---|---|---|
| `investment` 투자유치 | K-스타트업 도약·스케일업 트랙 "누적 투자 X억 이상", TIPS 운영사 투자 요건. 현행 룰셋이 kstartup-spine이므로 준핵심 | 누적 투자유치액, 최근 라운드·시기, TIPS 운영사 투자 여부 |
| `premises` 사업장·시설 | 지자체 "관내 사업장" 판정에 본점/지점 구분 필요(현 region은 주소 1개). 공장등록·자가/임대·입주 가능 여부 | 본점 외 사업장 소재지 목록, 공장등록 여부, 자가/임대 |
| `export_performance` 수출 실적 | 수출바우처 계열은 수출액 **구간**(내수→수출강소)이 자격·한도 결정. boolean 아님 | 직전년도 수출액(직/간접), 수출 이력 |

온라인 판매채널(통신판매업 신고 등)은 빈도가 낮아 차원 승격 보류 — certification canonical 사전(신고·등록증)으로 흡수.

### 5.3 기존 차원의 값 스키마·프로필 확장 (6건)

- **prior_award**: "이전 수혜 목록"에서 → (사업명·주관부처·연도·**진행상태**) 구조화. "동일사업 중복수혜 배제"와 "유사과제 **현재 참여 중** 배제", 사관학교류 프로그램 수료 이력까지 커버.
- **biz_age**: 재창업 여부, 과거 폐업 이력, 개인→법인 전환(업력 승계) — 재도전 사업·업력 상한 판정 오차의 원천.
- **industry**: 주업종/부업종 구분 + **공통 배제업종 canonical set**(유흥·사행·부동산·암호화자산 등) 일회 정의 → 공고마다 other 원문으로 반복되는 배제업종 조건을 `not_in`으로 정규화.
- **target_type**: 법인격 세분 — 협동조합/사회적협동조합/농업회사법인/비영리/단체.
- **certification**: canonical 사전을 인증(벤처·이노비즈)을 넘어 확인서·등록·신고(뿌리기업, 소부장, 기업부설연구소, 관광사업 등록, 사회적기업)까지 확장.
- **founder_age/trait**: 연령이 아닌 **생년월일**로 수집(공고마다 "만 39세" 기준일이 상이), 성별·장애 여부.

## 6. 차원으로 만들지 않는 것

1. **성격 축(필수/우대/결격)**: `kind`가 이미 커버. 추가 불필요.
2. **개인 대상 공고 조건**(직급·재직기간 등): 상류 audience 게이트로 해결 (§3.3).
3. **절차·재량 조건**(서류 허위, 표절, "기타 부적합"): 판정 불가 — other 잔존이 정답.
4. **제도별 상시근로자 산정 차이**: 차원이 아니라 **원천값(월별 인원, 피보험자 수) 저장 + 파생 계산**으로 대응.

## 7. 사용자 질문지 — 이론상 필요한 전체 수집 항목

신규 결격 4축(체납/신용/제재/중복수혜)은 전부 **예/아니오 수준의 저부담 문항이면서 판별력은 최상위** — 해당자에게는 불가 사업을 통째로 거르고, 비해당 대다수에게는 conditional로 새던 공고를 eligible로 확정시킨다. 고부담 문항은 financial_health(결산 수치)뿐이다.

| 차원 | 문항 | 형태 |
|---|---|---|
| tax_compliance | 국세/지방세 체납이 있나요? → (있다면) 납부유예·징수유예 승인을 받았나요? | 예/아니오 ×2 |
| credit_status | 금융기관 연체·채무불이행 / 회생·파산·법정관리 진행(+변제 정상이행) / 자산 압류 | 예/아니오 ×3 |
| sanction | 정부지원사업 참여제한·제재 중(종료일) / 부정수급·환수 이력 | 예/아니오 ×2 (+날짜) |
| financial_health | 최근 결산 부채총계·자본총계·자본금·자산총액 | 숫자 ×4 (또는 재무제표) |
| insured_workforce | 고용보험 성립 / 피보험자 수 / 최근 12개월 감원 | 예/아니오+숫자+예/아니오 |
| investment | 누적 투자유치액 / 최근 라운드·시기 / TIPS 운영사 투자 | 숫자+선택+예/아니오 |
| premises (선택) | 본점 외 사업장 소재지 / 공장등록 / 자가·임대 | 목록+예/아니오+선택 |
| export (선택) | 직전년도 수출액(직/간접) | 숫자 |
| prior_award (확장) | 수혜 이력: 사업명·부처·연도·상태 + 현재 참여 중 과제 | 구조화 목록 |
| biz_age (확장) | 재창업 여부 / 폐업 이력 / 법인전환 | 예/아니오 ×3 |
| industry (확장) | 주업종/부업종 + 매출 비중 | KSIC 선택 |
