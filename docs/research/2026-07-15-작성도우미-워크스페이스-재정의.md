## 0. 페이지 목적 재정의 (헌법 제1조)

이 화면이 답하는 사용자 질문은 **단 하나**다:

> **"이 칸에 이 값을 넣어도 되나요?"**

작성 도우미는 에디터가 아니고, 채팅 앱이 아니고, 문서 관리자도 아니다. **신청서의 다음 한 칸을 확인하는 루프**다. 좌측에서 "지금 어느 칸을 채우는지" 보이고, 우측에서 그 칸 하나를 확인하고, 확인이 끝나면 완성된 HWPX를 내려받는다. 그게 전부다.

- 엔진(서식 분석·프로필 시드·LLM 제안·label 충돌 감지·사다리 판정)은 복잡해도 된다. 조종석에 보이는 것은 **미리보기 + 카드 1장 + 진행률** 3가지뿐이다.
- 성공 기준(스펙 §8 우선순위 그대로): "칸마다 설명과 추천값이 붙고, 승인하면 좌측 표에 값이 꽂힌다"가 눈으로 체감되면 성공.

## 2. 결정 — 유지 / 변경 / 제거

### 유지 (그대로)

- **PreviewCanvas + 오버레이 양방향 동기화** — 이 화면의 심장. 셀 3상태(민트 체크/블루 하이라이트/옅은 회색)는 스펙과 이미 일치
- **낙관적 PATCH 규약** (`patchAnswer`의 필드 단위 반영·롤백, 교차-필드 클로버 방지) — 서버 규약 변경 없음
- **`하나씩 | 전체 목록` 토글** — 스펙이 허용한 유일한 보조 뷰
- **사다리 (b)** — "작성 항목을 분석하고 있습니다" 고지 + missingFields 질문 카드. 이미 미니멀
- **완료 카드** (`모든 항목 확인 끝!` + HWPX 다운로드) — 스펙 변형 C와 일치
- **중복 label 경고** — 정직성 원칙. 단, 뱃지가 아니라 카드 본문 한 줄로
- **ConversionPollTrigger** — 기계 배선, UI 아님

### 변경 (감량)

**① 상단 바 → 3요소로.** `← 공고 요약` / 공고명 / **`M/N 확인 완료` 단일 진행 표시** + 토글. LADDER_BADGE 삭제. 필수/전체 이중 표기 폐기 — 연결된 필드는 전부 "확인 대상"이므로 축은 하나면 된다. 문서가 2개 이상일 때만 문서 Select가 상단 바에 들어온다(푸터에서 이동).

**② FieldCard → 스펙 구조 그대로 재구성.** 위에서 아래로, 슬롯 7개 고정:

1. 상단 라벨: `확인이 필요한 항목 {N}개 중 {M}번째` — N = **미확인(empty+reviewing) 필드 수**, M = 그중 현재 카드 순번. 확인할수록 N이 줄어드는 게 보상
2. 필드명(굵게) + **위치 캡션**: `신청서 {page}쪽{section ? ` · '{section}' 표` : ""}` — `parsePositionPage`로 이미 계산 가능. position 없으면 캡션 생략(뱃지 아님)
3. **설명 한 줄** (신설 슬롯): 데이터 우선순위 — ⑴ lesson tip 첫 문장 ⑵ `mappedCompanyField` 기반 결정론 문구(예: 매출액 → "직전 회계연도 매출액을 적는 칸이에요") ⑶ 없으면 슬롯 생략. LLM 칸 설명 생성은 **후속 엔진 과제**로 분리 — 이번 감량에서 새 API를 만들지 않는다
4. 값 박스: 제안값(큰 타이포, 블루 틴트) + 출처 캡션(`{basis} 기준`). 값 없으면 변형 B(텍스트 입력 + 힌트 + `초안 제안 받기` 링크)
5. **주 CTA 1개**: `이 값으로 채우기` / `입력 완료` / `다음 항목 확인하기` (상태별 하나)
6. 보조 링크 2개: `직접 수정` / `건너뛰기` — 이 둘뿐. ⋯드롭다운 삭제: `초안 제안 받기`는 변형 B의 본문 링크로 표면화, `제안 값으로 되돌리기`는 직접 수정(편집) 상태의 보조 링크로 이동
7. 하단 한 줄: `💬 이 항목이 궁금하면 물어보세요`

삭제되는 뱃지: `필수`(연결 필드는 전부 확인 대상 — 구분이 행동을 바꾸지 않음), 상태 뱃지(값 박스와 진행 표시가 이미 상태를 말함), `위치 미확인`(캡션 생략으로 표현). 중복 경고만 카드 본문 경고 한 줄로 유지.

**③ 카드 아래 확인 완료 축약 리스트** (신설): `✓ 상호명 · ✓ 사업자등록번호 · ✓ 대표자명 …` 한 줄 텍스트(truncate). 클릭 요소 아님 — 다시 보려면 전체 목록 토글.

**④ 채팅 → Sheet 오버레이.** `이 항목이 궁금하면` 클릭 시 필드 패널을 대체하지 않고 우측 Sheet(shadcn)로 ChatPanelView를 띄운다. 닫으면 루프가 그 자리에 그대로 있다. 프리필(ADR-9)·컨트롤러(`useGrantChat`)는 그대로 재사용.

**⑤ 푸터 삭제.** `WorkspaceFooter` 컴포넌트 제거. 이동 경로:
- 문서 Select → 상단 바 (2개 이상일 때만)
- `WorkspaceDownloadButton` → 완료 카드(주 CTA) + **전체 목록 뷰 하단 보조 버튼**(중도 다운로드 니즈 — outline variant, "지금까지 확정한 값으로 내려받기")
- `hwpxTemplateAvailable=false` 고지 → 완료 카드에서만

### 제거 (이 화면에서 퇴출)

**⑥ DraftFallbackEditor를 사다리 (c)에서 뺀다.** (c) = 변환 전/실패/.doc·웹폼으로 연결 필드 0건인 공고. 이 경우 화면 8의 목적("이 칸 확인")이 성립하지 않으므로, (c) 화면은 다음 3가지만 남긴다:

1. 정직 고지 (`honestNotice` — 유지)
2. ChatPanelView (전면 — 유지, 초안 문안이 필요하면 채팅이 답한다)
3. 기관 연락처 카드 (`institutionContact` — 유지)

`DraftFallbackEditor`(937줄)와 그 5개 부속 기능(마크다운 편집·DOCX/MD/HTML 내보내기·섹션 재생성·품질 피드백·문항 편집)은 렌더 트리에서 제거한다. **파일은 삭제하지 않고 보존** (샌드박스 unlink 차단 + 재티어링 무통증 복귀 관례 — Pro/Flex `isActive=false` 선례). → 열린 결정 ①

- 카드 상태 변형: **A 자동값 확인**(위) / **B 직접 입력**(입력 + 힌트 1줄 + `초안 제안 받기` 링크 + CTA `입력 완료`) / **완료**(축하 카드 + `함께 완성한 신청서 내려받기 (HWPX)` + 보조 `전체 항목 다시 보기`)
- 전체 목록 뷰: 필드명 + 3상태 점 + 값 한 줄 × N행 + 하단 보조 다운로드. **그 이상 금지**
- 채팅: Sheet 오버레이. 루프 위에 얹히고, 닫으면 사라진다

### 사다리 (b) — 필드 분석 중

현행 유지: 프리뷰 + "작성 항목을 분석하고 있습니다" 고지 + missingFields 질문 카드(입력 → 초안 재생성). 진행률·토글은 숨김(현행과 동일). LADDER_BADGE만 빠진다.

### 사다리 (c) — 필드 확인 루프 불성립

정직 고지 + 채팅 전면 + 기관 연락처. §2-⑥ 참조. 편집기 없음.

### 모바일 (390px)

프리뷰를 상단 1/3 고정(현재 셀 중심 — 기존 `h-52` 크롭 유지), **인터뷰 카드가 주인공**. 채팅은 데스크톱과 동일하게 Sheet(전면). 푸터 없음.

### 금지 (스펙 계승 + 이 문서 추가)

- 필드마다 버튼 5~6개 나열 · 필드 3상태(확인 완료/확인 중/미입력) 외 상태 어휘 발명 · 3패널 동시 노출 · 전체 필드를 폼 그리드로 펼치기
- (추가) 사다리·ladder·draft 등 내부 개념의 화면 노출 · 상시 다운로드 버튼 · 채팅의 필드 패널 대체
