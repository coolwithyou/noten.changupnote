## 1. P1 — 모델·비용·처리량 (라이브 확인)

- **가격**(https://platform.claude.com/docs/en/about-claude/pricing.md [fetch 확인]):
  Opus 4.8 $5/$25(/MTok) — extractor.ts 단가 상수 3종과 정확히 일치. Sonnet 5는
  **인트로 $2/$10(2026-08-31까지)** 후 $3/$15, Haiku 4.5 $1/$5(컨텍스트 200K — 최대 공고
  100,851 입력 토큰 수용 가능하나 여유 얇음). Fable 5는 API 특성(분류기·보존 요건)상 배치
  추출용 부적합. 파일럿 실측(입력 5,978~100,851 토큰, $0.115~0.642/공고)과 정합 재확인.
- **Batches API**(https://platform.claude.com/docs/en/build-with-claude/batch-processing.md
  [fetch 확인 + 감독자 재확인]): 50% 할인($0.395→~$0.20/공고), 대부분 1시간 내·최대 24h,
  forced tool_choice 지원 — extractor 요청 본문 그대로 사용 가능. **판정: 실험 단계($40,
  실병목은 검수 노동)에선 폴링·custom_id 매칭 코드 신설의 손익이 안 맞음. 운영 전환
  (open ~1,500건 상시 분석, 회당 ~$600 규모) 시 기본값.**
- **Rate limit**(https://platform.claude.com/docs/en/api/rate-limits.md [fetch 확인]):
  최저 티어도 Opus 합산 1,000 RPM·입력 2M 토큰/분 — 동시성 3×100K 토큰이 한도의 ~8%.
  동시성 2~3은 과잉 보수 수준으로 안전. 단 신규·저사용 조직은 낮은 한도로 시작할 수 있음
  (첫 실행 429의 예상 원인 — 계획의 "소량 5건 확인" 유지).
- **타사 모델**: GPT-5.6 패밀리 가격은 서드파티 집계만 [미확인 — 공식 페이지 열람 실패],
  Gemini 3.1 Pro $2/$12(≤200K)·배치 50% 동일(https://ai.google.dev/gemini-api/docs/pricing
  [fetch 확인]). 한국어 장문 구조화 추출의 독립 벤치는 부재 — 멀티벤더는 tool use·span
  검증 파이프 재구현 비용까지 감안해 후보 등재만("측정 전 채택 금지").

## 2. P2 — 프롬프트·추출 기법

- **정합 확인**: 근거 span 강제 + 서버 측 실재 검증은 Google LangExtract(2025,
  https://github.com/google/langextract [fetch 확인])의 "정확 오프셋 grounding"과 같은
  계열이며, 검증 주체가 모델이 아닌 서버라는 점에서 더 보수적. forced tool_choice 스키마
  강제도 표준 관행.
- **다단계 vs 단일 패스**: 다단계 우위 문헌(arXiv 2510.05142, 프로덕션 회고들)의 성립
  조건은 수백 페이지급 초장문 — 우리 입력(대부분 1~3만 토큰)과 파일럿 실측(정밀도 96.4%)
  에서 성립하지 않음. self-consistency k회 다수결은 임상 IE에서 F1 +0.1~0.3 전례(arXiv
  2306.00024 [fetch 확인])가 있으나 비용 k배 — STOP/ITERATE 시 원인 층 한정 시험 후보.
- **실재하는 위험 1건**: lost-in-the-middle·장문 recall 저하(arXiv 2511.13900 등)는 최대
  입력 ~100K 토큰 구간에 실제로 걸린다. 파일럿의 "누락 0"은 두꺼운 공고 1건의 실측이라
  이를 통계적으로 배제하지 못함 → **집계에 "누락·ambiguous의 원문 내 위치(전/중/후반)
  분해" 진단 지표를 착수 전 등록**(판정 규칙 아님·진단 전용, §5 반영).

## 3. P3·P4 — taxonomy 승격·게이트 수치

- **P3**: LLM 스키마 진화 연구(AdaKGC·PARSE 등 [검색 확인])에 정량 승격 표준 없음. 가장
  가까운 정착 관행은 질적 연구의 코드 포화 — 신규 코드는 ~9단위에서 포화(Hennink 2017
  [검색 확인]), 중단 규칙은 "연속 2~3단위 신규 0건"(빈도 임계가 아니라 신규 출현 소진).
  "공고 간 독립 반복 후 승격"은 이 관행과 정합. **승격 규칙을 내부 정의로 사전 등록**(§5):
  재실행 제외 독립 공고 ≥3건 + 기존 축·text_only로는 판정 결과가 달라지는 기업이 존재
  (리트머스) + evaluator·질문·소싱 실현성. "≥3"은 외부 표준이 아닌 내부 정의임을 명기.
- **P4**: 직접 등가 외부 준거 부재(정부 지원사업 도메인의 codifiable % 보고 없음 — RaC
  문헌은 정성 서술뿐). 인접 임상 문헌의 구조화 가능 비율은 정의에 따라 23~74%(Ateya 2016
  74% [fetch 확인] / Raghavan 2015 23~41% [fetch 확인] 등) — 50%는 그 중앙부로 공격적이지도
  보수적이지도 않다. 게이트 정당화는 내부 목적 정합성(판정 문서 §6)에 두고, 외부 수치는
  분모가 달라 인용 금지. 회색지대 "판정 불능" 경로가 이미 사전 등록된 것이 설계 강점.

## 4. P5·P6 — 검수 운영·통계 설계

- **앵커링은 실재 위험**: LLM 제안 노출이 라벨 분포를 제안 쪽으로 이동시키고 속도는 안
  올림(크라우드워커 410명 사전등록 실험, arXiv 2507.15821 [fetch 확인]); 플래그 처리에
  노력이 들면 부정확한 제안 수용이 증가(2,784명, arXiv 2509.08514 [fetch 확인]). 파일럿의
  correct 쏠림(27/28)이 진짜 품질인지 앵커링인지 소표본으로는 구분 불가 — 우리 우려와 정합.
- **이의 플래그는 조건부 지지**: 모델 critique가 사람이 놓친 결함 발견을 돕고(Saunders
  2022 [검색 확인]), "판단 먼저, AI 나중" cognitive forcing이 과의존을 유의하게 감소
  (Buçinca CSCW 2021 [검색 확인]). 단 "이의 플래그 그 자체"의 직접 실증은 미발견(인접
  과제 외삽). **LLM-as-judge 보조 검수를 리뷰어 베타에서 확대 실험로 앞당김 — 조건부
  채택**: ① 확정 권한 없음(순환성 가드 — contract.ts 원칙) ② **사람 판정 저장 후에만
  노출**(사후 노출) ③ 플래그는 재검수 우선순위 신호로만. 플래그 LLM은 분석 모델(Opus)과
  **다른 모델**로(judge 자체 편향 경고, arXiv 2604.16790 [검색 확인]).
- **시차 재검수**: 10% 표본·κ 기준은 관행 범위(관대한 쪽). chart abstraction 관행
  [검색 확인]은 **재판정 시 원판정 비접근**을 명시 절차로 둠 — 계획에 절차로 추가(§5).
- **P6 구조 정합**: 30(진단·중단만)→100(확증) + 사전 등록은 Simon two-stage와 동형.
  형식적 알파 통제는 목적 혼합 설계에 과설계 — 사전 등록으로 갈음 합리적. 클러스터
  부트스트랩+ICC 실측은 관행 정합(소클러스터 under-coverage 유의 — 불안정 시 Rao-Scott
  조정 Wilson이 대안 후보).
- **순환(double dipping) 1건 발견·수정**: "100건 최종 집계에 파일럿 3건 병합"은 게이트
  수치(50%)를 유도한 데이터가 판정 표본에 재진입하는 구조(Kriegeskorte 2009 [검색 확인]의
  circular analysis). 영향은 미미(27/~900)하나 원칙 위반 — **게이트 판정은 파일럿 제외
  표본으로, 병합치는 민감도 참고로만 병기**(집계 코호트 필터의 기본 동작이 이미 제외라
  구현 변경 불요, 계획 문구만 수정). **동결(프롬프트·가이드·판정 규칙)은 커밋 해시를
  명시**해 사전 등록을 실효화.

## 5. 개선 후보 통합표와 계획 반영

**확대 실험 착수 전 도입(전부 저비용, 계획 문서에 반영 완료)**:

| # | 항목 | 비용 | 반영 위치 |
|---|---|---|---|
| 1 | 이의 플래그 **사후 노출** 3조건 + 타 모델 judge | 절차 명문화(+구현 소폭) | 계획 §4 완화 4번 |
| 2 | 게이트 판정 표본에서 **파일럿 3건 제외**(병합치는 민감도 병기) | 문구 수정(필터는 기구현) | 계획 §3 |
| 3 | **동결 커밋 해시 명시**(프롬프트 v2·가이드 §5·판정 규칙) | 극저 | 계획 §3 |
| 4 | **축 승격 규칙 사전 등록**(독립 공고 ≥3 + 리트머스 + 실현성) | 극저 | 계획 §3 신규 소절 |
| 5 | 시차 재검수 **원판정 비접근** 절차 | 저 | 계획 §4 완화 1번 |
| 6 | **위치별 누락 진단 지표**(전/중/후반 분해) | aggregate +30~50줄 | 계획 §5 잔여 구현 |

**확대 실험 이후 후보(등재만 — 측정 전 채택 금지)**: Batches API 전환(운영 시 기본값,
비용 50%↓), Sonnet 5 A/B(env 변경만·재분석 ~$5/30건 — **인트로 가격 8/31 종료 유의**,
run.model 필드로 병합 오염 없음), strict:true 스키마(minItems 호환 미확인), self-consistency
다수결(STOP/ITERATE 시 원인 층 한정), seeded error 감사(κ 나쁠 때), Rao-Scott CI(부트스트랩
불안정 시).

**기각·금지**: 프롬프트 캐시(고정 프리픽스 ~5.5K 토큰뿐 — 100건 ~$2.5, 효과 미미),
LLM 플래그의 확정 권한 승격(순환성 가드 위반 — 영구 금지).

## 6. 불확실성·한계

- 앵커링 연구 2건은 크라우드워커·주관 과제 기반 — 도메인 전문가(창업자)로의 외삽이며,
  전문가는 덜 취약할 수 있다는 반대 가설도 기각되지 않음.
- P4 외부 수치는 전부 임상 도메인·상이한 분모 — 범위 참고용.
- [미확인]으로 남은 것: GPT-5.6 공식 가격, strict 모드와 minItems/maxItems 충돌 여부,
  코퍼스 라벨 추가 임계("≥10회/≥5문서") 원문, HDSR 게재본(403). 인용하지 않았다.
- LLM-as-judge "production 관행" 서술 다수는 벤더 콘텐츠 — 근거 가중치 낮게 처리했다.
