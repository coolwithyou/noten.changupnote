# 기업 데이터 확장 연구 — 매칭 정확도 향상과 확인서 발급 가능성 사전 판정

> 작성일: 2026-07-10
> 질문: "기업의 모든 정보에 접근 가능하다고 가정할 때, 어떤 정보가 있으면 지원사업 매칭 정확도를 확실히 높일 수 있는가? 그리고 창업기업확인서·소상공인확인서를 필수 요구하는 사업(예: LIPS)의 발급 자격 사전 판정을 위해 팝빌 외에 반드시 확보해야 할 데이터는 무엇인가?"
> 근거: 코드베이스 전수 탐색(스키마·매칭 엔진·팝빌 연동) + 외부 도메인 리서치(확인서 발급 요건·공공 API 소스, 2026-07 기준)

---

## 0. 요약 (TL;DR)

1. **현재 매칭 14축 중 팝빌이 실질 커버하는 축은 5개**(region, biz_age, industry, size 근사, business_status)이고, 나머지 9축(revenue, employees, founder_age, founder_trait, certification 대부분, prior_award, ip, target_type 일부)은 자가신고 의존 → unknown → `conditional` 판정으로 새기 때문에 매칭 정확도의 병목이 **공고 파싱이 아니라 기업 프로필의 결손**에 있다.
2. 확보 용이성 대비 판별력이 가장 높은 데이터는 ① **상시근로자 수**(국민연금·고용산재 무료 공공 API, 무인증) ② **벤처확인 등 인증 보유**(무료 공개 조회) ③ **매출액**(사용자 동의 스크래핑 또는 자가신고) ④ **대표자 과거 사업이력**(사실증명 총사업자등록내역 — 창업기업 판정의 유일한 원천).
3. 창업기업확인서·소상공인확인서는 **원천 데이터로부터 발급 가능성을 규칙으로 사전 판정할 수 있다**. 소상공인확인서는 "업종별 평균매출액 기준 + 상시근로자 5/10인 미만" 2단 판정이라 Tier 1~2 데이터만으로 거의 확정 판정 가능. 창업기업확인서는 "업력 7년"은 팝빌로 되지만 **"창업으로 보지 않는 경우"(폐업 후 동종업종 재개업, 법인전환, 승계)** 판정에 대표자 명의 전체 사업자등록 이력이 반드시 필요하다.
4. 스키마는 이미 준비되어 있다: `company_profiles.source` enum에 `codef`/`ocr`이 예약되어 있고(현재 미사용), `grant_criteria`의 `certification` 축과 canonical 인증 사전이 확인서 이름을 이미 안다. 필요한 것은 **새 dimension이 아니라 새 source 커넥터와 파생 판정(확인서 발급 가능성) 레이어**다.

## 1. 현재 상태: 14축 × 데이터 원천 커버리지

매칭 엔진(`packages/core/src/matching/match.ts`, ruleset-kstartup-spine-v2)은 `grant_criteria`의 축별 조건을 `CompanyProfile`과 대조해 pass/fail/unknown으로 채점한다. unknown이 하나라도 있으면 `conditional`(60~95점)로 강등되므로, **프로필 결손 축 하나가 곧 점수·신뢰 게이트 하락**으로 직결된다.

| 축 (dimension) | 현재 원천 | confidence | 결손 내용 |
|---|---|---|---|
| region | 팝빌 `addr` 시도 매칭 | 0.8 | 본점 vs 사업장 구분 없음. 지자체 사업은 "사업장 소재지" 기준이 많음 |
| biz_age | 팝빌 `establishDate` | 0.75 | 개업일 기준일 뿐, **재창업·법인전환 시 업력 승계** 미반영 |
| industry | 팝빌 `industryCode`(KSIC)+업태·종목 | 0.6~0.7 | 주업종/부업종 구분 없음, 세세분류(5자리) 비교 불가 케이스 존재 |
| size | 팝빌 `corpScaleCode` | 0.65 | 국세청 규모코드는 소상공인/소기업 법정 판정과 불일치 가능(매출·근로자 미반영) |
| revenue | **자가신고만** | — | 공고의 매출 상·하한 조건에서 unknown 다발 |
| employees | **자가신고만** | — | 소상공인 5/10인, 고용요건 조건에서 unknown 다발 |
| founder_age | **자가신고만** | — | 청년(만 39세 이하) 사업 판정 불가 |
| founder_trait | SMPP 여성·장애인기업 확인서 (positive-only) | — | 미보유 시 성별·장애 여부 자체는 알 수 없음 |
| certification | SMPP 2종 자동 + 자가신고 | — | 벤처·이노비즈·메인비즈·연구소·사회적기업 등 나머지 15종 자동조회 없음 |
| prior_award | 자가신고만 | — | 동일사업 기수혜 제외 조건 판정 불가 |
| ip | 자가신고만 | — | 특허 보유 우대 조건 판정 불가 |
| target_type | kind(active/preliminary)+추론 | — | 예비창업 여부만 |
| business_status | NTS 상태조회 + 팝빌 `closeDownState` | 0.8 | 양호 (무료 게이트로 신선도 유지) |
| other | text_only | — | 원문 검수 대상 |

**진단**: business_status·region·biz_age·industry는 준수. **revenue·employees·certification·founder_* 4개 영역이 unknown의 대부분을 생산**하며, 이 4개가 공고 자격요건에서 가장 빈출하는 축이기도 하다(규모·업력·인증·대표자 특성은 사실상 모든 공고에 등장).

## 2. 매칭 정확도를 "확실히" 높이는 기업 정보 (우선순위)

기준: (공고 조건 빈출도 × unknown 해소 폭) ÷ 확보 난도.

### P1 — 상시근로자 수 (즉시, 무료, 무인증)
- **효과**: employees 축 + size 축(소상공인/소기업 판정의 절반) + 고용요건 조건 동시 해소. 소상공인 대상 사업(소진공 계열 전체, LIPS 포함) 필터의 핵심.
- **원천**: 국민연금 가입 사업장 내역(공공데이터포털 3046071, 사업장별 가입자 수·월 신고액) + 근로복지공단 고용·산재보험 현황(15059256, 상시근로자 수·보험 성립일). 둘 다 **사업자번호만으로 조회 가능한 무료 API**.
- **주의**: 국민연금은 법인 3인 이상/개인 10인 이상만 노출(2025.7 기준 개편) → 소규모 사업장은 고용산재 API로 교차, 둘 다 없으면 "가입자 0~미확인"을 소상공인 방증(confidence 낮음)으로 활용.

### P2 — 인증 보유 자동조회 확대 (즉시, 무료)
- **효과**: certification 축 unknown 대량 해소. 현재 SMPP 2종만 자동인데, 아래는 전부 무료 공개 조회다:
  - **벤처기업확인**: 벤처확인종합관리시스템 공시 검색(smes.go.kr/venturein) + 공공데이터포털 벤처기업명단 파일(15084581). 확인서 발급일·유효기간까지 공시 의무.
  - **사회적기업**: 고용부 인증현황 공공데이터(15090102).
  - **기업부설연구소/전담부서**: KOITA 신고관리시스템 검색(rnd.or.kr).
  - **이노비즈/메인비즈**: 각 넷 조회(스크래핑 필요, 구조 단순).
- **구현 결**: 기존 SMPP 커넥터(`packages/core/src/smpp/check-certificates.ts`)와 동일한 positive-only·30일 캐시 패턴으로 병렬 확장.

### P3 — 매출액 (사용자 동의 필요, 판별력 최상)
- **효과**: revenue 축 + size 축 완결(소기업 = 업종별 평균매출 10억~120억 이하). 소상공인·중소기업확인서 발급 가능성 판정의 핵심 입력.
- **원천**: 부가가치세과세표준증명(홈택스) — 공식 API 없음. **CODEF/하이픈 등 인증 대행 스크래핑**(사용자 홈택스 인증 동의) 또는 유료 CB(NICE BizAPI, KoDATA 크레탑 — 개인사업자 포함 재무).
- **폴백**: 자가신고 매출 구간 입력(현행) + 추후 증빙 업로드 OCR 검증. `source` enum의 `codef`·`ocr`이 정확히 이 자리다.

### P4 — 대표자 과거 사업이력 (창업기업 판정의 유일 원천)
- **효과**: biz_age 축의 질적 완성 — "업력 7년 이내"가 아니라 **"법상 창업기업인가"**를 판정. 재창업·폐업 후 동종업종 재개업·기존사업 병행(연쇄창업) 케이스를 걸러낸다. §3 참조.
- **원천**: 국세청 **사실증명(총사업자등록내역)** — 대표자 명의 전체 사업자등록의 개업일·폐업일·업종코드. 홈택스 스크래핑(사용자 동의) 또는 서류 업로드+OCR만 가능. 공공 API 없음.

### P5 — 대표자 생년·성별
- **효과**: founder_age(청년/시니어 사업)·founder_trait 축 해소. 청년창업사관학교, 청년 전용 자금 등 대형 트랙 판별.
- **원천**: 회원 본인인증(PASS/아이핀) 시 생년 확보가 최저비용. 법인은 법인등기부(대표자 생년월일 기재)로 교차 가능. 자가신고 폴백.

### P6 — 특허·지식재산 (무료 쿼터 API)
- **효과**: ip 축 해소(기술창업 사업 우대·가점 조건). **원천**: KIPRIS Plus API(출원인명 검색, 기본 무료 쿼터). 출원인명≠상호 동명이인 문제가 있으므로 confidence 중간으로.

### P7 — 국세·지방세 체납, 채무불이행 (배제 조건 전용)
- **효과**: 거의 모든 공고의 공통 배제 조건("국세·지방세 체납 시 제외") 판정. 매칭 축으로는 business_status/other에 얹힘.
- **원천**: 납세증명(홈택스·위택스) — 사용자 동의 스크래핑 또는 업로드. 신용 채무불이행은 CB 유료 영역이라 자가신고+협약 시 서류 검증으로 충분.

### P8 — 본점/사업장 소재지 분리, 주업종·부업종 구분
- 팝빌 응답에 이미 있는 주소·업종 데이터의 **정밀화**(신규 수집 불필요). 지자체 사업 region 오탐과 industry 세세분류 비교 실패를 줄인다. 법인은 법인등기부(본점)와 사업자등록(사업장)을 구분 저장.

> **의도적으로 후순위**: prior_award(정책자금 기수혜)는 공적 조회 창구가 없어(기관 내부 시스템) 자가신고+공고별 확인이 상한선. 임베딩 유사도 매칭(`grants.embedding` 미사용 컬럼)은 프로필 결손이 해소된 뒤에야 효과가 있다.

## 3. 확인서 발급 가능성 사전 판정 — "이 서류, 발급 받을 수 있나요?"

LIPS(기업가형 소상공인 육성, 소진공)처럼 **소상공인확인서를 필수 제출서류로 요구**하는 사업이 있고, K-Startup 창업사업화 공고는 사실상 창업기업 요건을 전제한다. 확인서 자체는 공적 판정의 결과물이므로, cunote가 원천 데이터로 **발급 가능성을 사전 판정**해주면 (a) 매칭 정확도 (b) "신청 전에 이 서류부터 발급받으세요" 액션 안내 두 가지가 동시에 잡힌다.

### 3.1 소상공인확인서 — 판정 규칙과 필요 데이터

발급: 중소기업현황정보시스템(sminfo) / 중소벤처24. 판정(소상공인기본법 제2조):

```
소상공인 = 소기업 AND 상시근로자 기준 충족
  ① 소기업: 주된 업종별 평균매출액 ≤ 중소기업기본법 시행령 별표3 기준
     (업종별 120억/80억/50억/30억/10억원 5구간)
  ② 상시근로자: 광업·제조업·건설업·운수업 10명 미만, 그 외 5명 미만
     (전년도 매월 말일 근로자 수 합계 ÷ 조업 개월 수, 월 60시간 미만·일용직 제외)
```

| 판정 입력 | 확보 경로 | 티어 |
|---|---|---|
| 주된 업종 KSIC (매출 비중 최대 업종) | 팝빌 industryCode + 부가세 신고 업종(스크래핑) | 보유+T2 |
| 평균매출액 (직전 3개년 또는 창업 후 기간) | 부가세과세표준증명·손익계산서 (CODEF/하이픈, 업로드 OCR) | T2 |
| 상시근로자 수 | 국민연금·고용산재 공공 API (근사) → 정밀은 원천징수이행상황신고서·건보 명부 | **T1**+T2 |
| 창업 연차 (직전·당해연도 창업은 무서류 발급) | 팝빌 establishDate | 보유 |

**시사점**: T1(무료 API) + 자가신고 매출만으로도 "발급 가능성 높음/애매/불가" 3단 판정이 가능하고, T2 연동 시 확정 판정. sminfo 자체가 홈택스 자료를 스크래핑으로 수집하므로 우리가 같은 데이터를 먼저 보면 결과를 그대로 예측할 수 있다.

### 3.2 창업기업확인서 — 판정 규칙과 필요 데이터

발급: 창업기업 확인시스템(cert.k-startup.go.kr, 창업진흥원). 판정(중소기업창업 지원법 제2조·시행령 제2조):

```
창업기업 = 중소기업 AND 사업개시 7년 이내 (신산업 분야 10년)
단, 아래는 "창업으로 보지 않음":
  ① 상속·증여로 타인 사업 승계 후 동종 사업 개시
  ② 기존사업 계속하면서 새 기업 설립 (위장·연쇄창업)
  ③ 폐업 후 동종 사업 재개 — 원칙 폐업 후 3년(부도·파산 2년) 미경과 시 불인정
     (2025-06 개정: 우수 재기역량 재창업자는 폐업기간 무관 인정)
  ④ 법인전환 후 동종 사업 계속 (단, 개인→본인 대표 법인 포괄이전은 잔여 업력 승계)
동종업종 = KSIC 세세분류(5자리) 동일 (2024 완화: 종전 세분류 4자리)
제외 업종: 일반·무도유흥주점업, 카지노, 기타 사행시설
```

| 판정 입력 | 확보 경로 | 티어 |
|---|---|---|
| 사업개시일·업력 | 팝빌 establishDate (개인) / 법인등기부 (법인) | 보유+T2 |
| **대표자 명의 과거·현재 전체 사업자등록** (개업일·폐업일·업종코드) | **사실증명(총사업자등록내역)** — 홈택스 스크래핑 또는 업로드 OCR. 공공 API 없음 | **T2 (필수)** |
| 동종업종 비교 | 과거 사업 vs 현 사업 KSIC 세세분류 대조 (위 사실증명 + 팝빌) | T2 |
| 법인 설립 경위 (전환·합병·분할) | 법인등기사항전부증명서(말소사항 포함) — 인터넷등기소 | T2 |
| 실질 지배관계 (위장창업) | 주주명부 — 공적 조회 불가, 업로드만 | T4 |
| 제외 업종 | 팝빌 industryCode | 보유 |

**시사점**: "업력 7년 이내 + 제외업종 아님"까지는 현재 데이터로 즉시 판정된다. 그러나 **불인정 사유(②③④)는 사실증명 없이는 원천적으로 판정 불가**이고, 실제 발급 반려의 대부분이 여기서 난다. 초기엔 "업력 통과 + 아래 3개 문항(과거 폐업 이력 있나 / 다른 사업 운영 중인가 / 법인전환인가) 자가응답"의 하이브리드가 현실적이고, T2 연동 시 자동 확정.

### 3.3 중소기업확인서 — 판정 규칙과 필요 데이터 (참고)

```
중소기업 = ① 3개년 평균매출액 ≤ 별표1 업종별 기준 (400억~1,500억)
        AND ② 자산총액 < 5,000억
        AND ③ 독립성: 자산 5천억 이상 법인의 30% 이상 피출자 아님,
              관계기업(30% 지배) 평균매출 지분비율 합산 후에도 기준 충족
졸업 유예: 규모 초과 후 5년 (2024-08 확대, 3년→5년)
```

필요 데이터: 3개년 재무제표(매출·자산), 조정후수입금액명세서(주업종), 원천징수이행상황신고서, **주주명부·법인등기부(관계기업 판정)**. 재무제표 미작성·원천세 미신고 창업기업은 무서류 발급 → **업력 1년 미만이면 "발급 가능성 높음"으로 즉시 판정 가능**한 지름길이 있다.

### 3.4 LIPS 팩트 확인

- LIPS = **기업가형 소상공인 육성 프로그램**(소상공인판 TIPS). 중기부 소관, 소진공 수행. LIPS I(민간 선투자의 최대 5배 매칭융자, 한도 5억) / LIPS II(최대 3배 사업화 보조금, 한도 2억) 2트랙.
- 신청 자격: 소상공인기본법 제2조 소상공인 + 신청일 현재 정상 영업 + 민간 운영사 선투자(확약).
- **필수 서류에 소상공인확인서 명시** (참가신청서·IR Deck·소상공인확인서·사업자등록증). 2026년 '소상공인 도약 지원(로컬기업육성·강한소상공인)' 재편 공고도 동일 계열.
- 매칭 관점: `f_required_certs` / `grant_criteria(certification, required)`에 "소상공인확인서"가 걸리는 공고군이며, 발급 가능성 사전 판정이 없으면 이 공고들은 전부 conditional로 새거나, 반대로 발급 불가 기업에 추천되는 오탐이 난다.

## 5. 구현 시사점 (스키마·엔진 관점)

1. **스키마 변경 최소**: `company_profiles`(EAV, source·confidence·as_of 보유)와 `company_enrichment_cache`(provider별 원본 캐시)가 그대로 수용한다. `source` enum의 `codef`/`ocr`은 이미 예약됨. Tier 1 신규 커넥터는 SMPP 패턴(`packages/core/src/smpp/`) 복제로 충분.
2. **파생 판정 레이어 신설**: "확인서 발급 가능성"은 원시 dimension이 아니라 **파생 판정**(소상공인 = f(업종, 매출, 근로자))이다. `packages/core/src/certification/`에 `eligibility-precheck.ts`류의 순수 함수로 두고, 결과를 `company_profiles(dimension='certification', value={cert:'소상공인확인서', status:'eligible_unissued'|'held'|'ineligible'|'unknown'})`처럼 상태 3분법으로 기록하면 매칭 엔진의 certsMatch가 required_certs 공고를 정확히 3단(추천/발급 안내 후 추천/제외)으로 가를 수 있다.
3. **next_question과의 결합**: 매칭 엔진의 `next_question`(unknown 축 1개 질문)이 이미 있으므로, T1 자동 수집으로 unknown을 줄인 뒤 남는 질문을 "창업기업확인서 불인정 사유 3문항" 같은 판정 직결 문항으로 교체하면 질문당 정보 이득이 최대화된다.
4. **confidence 정책**: 국민연금/고용산재 근로자 수는 근사치(가입 제외 인원 존재)이므로 0.6~0.7, 확인서 실물 보유는 0.95+, 자가신고는 0.5 유지 — 신뢰 게이트(`buildReviewGate`)의 recommendable 티어 산정과 일관되게.
5. **순서 제안**: T1 커넥터(국민연금·고용산재·벤처·사회적기업·연구소) → 소상공인 발급 가능성 프리체크(T1+자가신고 매출) → 본인인증 생년 확보 → T2(CODEF: 부가세 증명·사실증명) → 창업기업 프리체크 완성.
