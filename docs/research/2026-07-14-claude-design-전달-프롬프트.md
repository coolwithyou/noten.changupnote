# 창업노트 — 사용자 화면 인터랙티브 프로토타입 제작 지시서

공공 지원사업 매칭 서비스 **창업노트**의 사용자용 화면 14개를 하나의 연결된 인터랙티브 프로토타입으로 만들어줘. 이 문서 하나가 전체 스펙이다. §0(공통)을 모든 화면에 적용하고, 화면 1~14를 스펙대로 만든 뒤, 마지막 §8 산출물 목록의 프레임을 전부 생성해줘.

## §0. 공통 — 모든 화면에 적용

### 0.1 제품과 사용자

- **창업노트**: 사업자번호 하나로 받을 수 있는 공공 지원사업을 찾아주고 지원서 작성까지 안내하는 서비스
- 사용자: 소상공인·초기 창업자. 공고 문법(적격/우대/배제)에 익숙하지 않고, 바쁘고, 관공서식 화면에 지쳐 있음
- 디자인 철학: **"복잡한 엔진, 단순한 조종석"** — 화면은 결론과 다음 행동 하나만 크게 말하고, 근거·점수·상세는 사용자가 요청할 때만 펼친다. 토스의 "One thing per one page", TurboTax의 인터뷰 패턴이 레퍼런스
- 이 제품의 심장은 화면 3(매칭 결과)의 루프: **회사를 설명할수록 매칭 정밀도가 오르고 결과가 좁혀진다.** 자동으로 확인된 정보와 직접 채울 정보의 구분이 명확하고, 빈 곳을 채우고 싶어지게 만드는 것이 전체 디자인의 성패

### 0.2 비주얼 토큰

- 배경 흰색, 본문 거의 검정(#222 수준), 보조 텍스트 회색(#6B7280 수준)
- Primary 블루 **#3182F6** (hover #1B64DA), 연한 블루 틴트 #E8F3FF
- 민트 **#2BD4A8** (긍정·확정 신호), 민트 소프트 #E6FBF1
- 경고/마감임박 레드 #F04452
- 라디우스 크게: 카드 16px, 버튼 12px. 그림자는 아주 옅게. 여백 넉넉하게
- 타이포: 한글 산세리프(Pretendard 느낌). 헤드라인 굵고 크게, 본문 15~16px. 숫자는 tabular
- 톤앤매너: 해요체, 짧은 문장. 관공서 용어 금지 ("귀하", "적격" 대신 "지금 신청 가능")

### 0.3 상태 어휘 (전 화면 공통 — 이 어휘 외 상태 라벨 발명 금지)

공고 판정 뱃지 4종:

| 문구 | 색 |
|---|---|
| **지금 신청 가능** | 민트 배경 + 진한 민트 텍스트 |
| **답하면 확정** | 블루 틴트 배경 + 블루 텍스트 |
| **원문 확인 필요** | 회색 배경 + 회색 텍스트 |
| **이번엔 어려움** | 연회색 배경 + 흐린 텍스트 |

정보 항목 3상태: **자동 확인**(민트 체크 ✓) / **직접 입력됨**(블루 점 ◉) / **미입력**(회색 빈 슬롯 + 채우기 유도)

보조 표기 2종 (뱃지가 아님): 접수 예정 공고는 D-day 자리에 `7/21 접수 시작` 텍스트로 / "준비하면 열려요"는 섹션 이름으로만 등장

**금지 문구**: "지원 가능성이 높음", "합격 확률" 등 확률을 약속하는 표현 절대 금지. 점수(%)는 접힘 카드에 노출 금지 (매칭 정밀도 게이지는 예외 — 회사 프로필 수준의 지표라 허용)

### 0.7 요금제·이용권 (과금 모델 공통 규칙 — "표면은 두 카드")

- **표면 플랜은 둘뿐**: **Free**(무료로 시작하기)와 **멤버십**(월 9,900원). 대행사·기관·컨설턴트는 카드가 아니라 하단 "문의" 배너. 그 외 티어·등급 발명 금지
- **경계 문장 (모든 과금 카피의 대전제)**: 공고를 찾고 확인하는 건 전부 무료. 돈을 받는 건 두 가지뿐 — **공고를 놓치지 않게 챙겨드리는 일**(실시간 알림·마감 리마인더 = 구독)과 **신청서를 대신 써드리는 일**(신청서 도우미 = 횟수)
- **어휘 규칙 (엄수)**: `크레딧`·`토큰`·환율·보너스율 어휘 **절대 금지**. 사용자 단위는 **횟수(회)** 하나 — `신청서 도우미 매달 12회`, `남은 3회` (12회는 목업용 샘플 값 — 최종 수치 미확정)
- **도우미 1회 = 풀 패키지**: 서식 분석 + 초안 + 자동 채움이 한 번에 포함. 화면 8 안의 개별 액션(초안 제안·수정 등)에는 추가 과금이 없다
- **무료 범위**: 매칭·판정·근거 열람 전부 무료 / '확인 필요' 해소 질문 월 3회 / 신청서 도우미 가입 축하 2회 / 새 공고 주 1회 이메일 요약
- **과금 3규약**:
  1. **시작 전** 확정 고지: `도우미 1회 사용 · 남은 3회` (고정 차감이므로 "약" 없이 확정 표기)
  2. **소진 시** 모달: `횟수를 다 썼어요` — **이용권 팩(즉시)과 멤버십(회당 더 저렴)을 나란히 제안**, 결제창 직행 금지
  3. **시작 직후** 토스트: `도우미 1회 사용 · 남은 2회`
- **남은 횟수 위젯**: 로그인 화면 헤더 우측 `✍ 남은 3회` (0회면 주황 `0회`). 클릭 → 화면 10
- 금지: 고지 없는 차감, 한계비용이 있는 항목에 `무제한` 표현(질문 루프처럼 원가가 거의 0인 항목에만 허용), 확률·성과 약속과 가격의 결합("이거면 선정돼요" 류)

## 화면 1. 랜딩

스크롤 5섹션의 초단순 랜딩. 목표는 단 하나 — 사업자번호 입력. 화면 자체가 "30초면 끝난다"는 인상을 줘야 한다.

1. **히어로** (첫 뷰포트의 주인공)
   - 상단 작은 뱃지: `● 지금 신청 가능한 지원사업 1,284건` (초록 점)
   - 헤드라인 2줄: `사업자번호만 넣으면,` / `받을 수 있는 지원사업이 보여요` (두 번째 줄 일부 블루 그라디언트 강조)
   - 대형 입력 폼 1개: placeholder `000-00-00000` + 블루 버튼 `지원사업 찾기`
   - 폼 아래 캡션 한 줄(회색, 작게): `회원가입 없이 30초 · 입력 정보는 암호화돼요`
   - 이 외 요소 금지 (칩 나열, 보조 CTA 없음)
2. **실공고 프리뷰 밴드**: 캡션 `매주 40여 개 기관의 공고를 수집해요` + 카드 3장 가로 배치(공고 1·4·5). 카드당 3요소만: 제목 / D-day 뱃지 / 금액
3. **3단계 안내**: 제목 `공부 없이, 세 단계면 충분해요` + 번호 카드 3장 — ① `사업자번호 입력` "번호 하나면 회사 정보를 자동으로 불러와요" ② `맞춤 매칭` "지금 신청 가능한 사업과 답하면 확정되는 사업을 나눠 보여줘요" ③ `신청 코칭` "서류의 80%는 미리 채워드리고, 빠진 것만 물어봐요"
4. **마지막 CTA**: 헤드라인 `사업자번호 하나로 지금 바로 시작하세요` + 히어로와 동일한 입력 폼
5. **FAQ + 푸터**: 아코디언 4개(기본 모두 닫힘) — `정말 사업자번호만 넣으면 되나요?` / `회원가입을 꼭 해야 하나요?` / `어떤 지원사업을 다루나요?` / `비용이 있나요?`. 푸터는 한 줄(© 2026 바톤 + 이용약관 · 개인정보처리방침)

금지: 기능 나열 섹션, 통계 숫자 섹션, 아이콘 그리드, 두 개 이상의 서로 다른 CTA 문구.

## 화면 2. 회사 확인 다이얼로그 (화면 1 위 오버레이)

랜딩 위에 뜨는 확인 모달. 3상태를 각각 제작:

1. **로딩**: 스피너 + `사업자 정보를 확인하고 있어요` + `123-45-***90`
2. **확인** (메인): 타이틀 `『주식회사 바다상회』 회사가 맞으신가요?` + 보조 정보 한 줄 `123-45-***90 · 영업 중 · 부산` (중점(·) 구분 텍스트 한 줄, 뱃지 나열 아님) + 버튼 2개: 블루 `네, 매칭 결과 보기` / 고스트 `아니요, 다시 입력할게요`
3. **오류**: `등록된 사업자를 찾지 못했어요` + 안내 한 줄 + 버튼 `사업자번호 다시 입력`

## 화면 3. 매칭 결과 — 정밀도 루프 (★ 가장 공들일 화면)

"공고 목록"이 아니라 **"회사를 설명할수록 정밀도가 오르고 결과가 좁혀지는 루프"**로 보여야 한다.

레이아웃 (위→아래, 단일 칼럼 max-width 760px):

   - `회사 정보 새로고침` — 캡션 `국세청 최신 정보로 다시 확인해요` (누르면 스피너 + 완료 토스트 `최신 정보로 확인했어요 · 바뀐 항목 없음`)
   - 하단 작은 링크: `다른 사업자로 변경` (→ 화면 2 확인 플로우 재사용)
3. **데이터 연결** (동의를 요구가 아니라 보상으로 — 각 행: 토글 + 이름 + 보상 캡션):
   - `국세청 홈택스` — `연결하면 매출·업력이 자동 확인돼요` (연결 안 함 상태)
   - `4대보험` — `상시근로자 수가 자동 확인돼요` (연결 안 함)
   - `기본정보 자동 갱신` — `회사 정보가 바뀌면 자동으로 최신화해요` (켬)
   - 섹션 캡션: `연결한 정보는 매칭 판정에만 쓰여요 · 언제든 끌 수 있어요`
4. **멤버십**: 상태 라인 `멤버십 이용 중 · 다음 결제 8월 15일 · 월 9,900원` + `결제 수단 변경` + 작은 텍스트 링크 `해지하기` (숨기지 않되 주장하지 않는 크기). Free 사용자 변형: `Free 이용 중` + 업셀 1줄 `멤버십이면 공고 놓칠 일이 없어요 · 자세히 ▸`
5. **알림**: 토글 3개 — `실시간 맞춤 알림` (Free면 비활성 + `멤버십 기능` 뱃지) / `마감 리마인더` (동일) / `주 1회 이메일 요약` (전원 가능). 알림 토글은 이 화면에만 존재(다른 화면에 중복 배치 금지)
- 최하단, 작고 흐리게: `데이터 삭제 요청` · `회원 탈퇴`

**13B. 대표자 확인 시트** (별도 컷): 안내 `사업자등록증의 정보로 대표자임을 확인해요` + 필드 2개(대표자명 / 개업일) + CTA `확인하기` + 완료 상태 `대표자 확인됨 ✓ — 이제 회사 전용 정보를 자동으로 가져와요`. "소유권 검증"·"캐시" 같은 시스템 어휘 금지
- **해지 확인 시트** (별도 컷, 다크패턴 금지): `정말 해지할까요?` + 사실만 담백하게: `남은 도우미 7회는 60일까지 그대로 쓸 수 있어요` / `실시간 알림은 다음 결제일(8월 15일)까지 유지돼요` + 버튼 2개 동등한 크기: `해지하기` / `계속 이용하기` — 감정적 만류 카피 금지
- **탈퇴 확인 시트**: `탈퇴하면 저장된 매칭 결과와 작성 중인 신청서가 모두 삭제돼요` + 확인 입력 + `탈퇴하기`

## 화면 14. 이메일·알림 템플릿 ("놓치지 않기"의 실물)

멤버십이 파는 가치의 실체. 이메일 템플릿 2종을 프레임으로 제작 (600px 폭 이메일 레이아웃):

- **주간 다이제스트** (Free, 매주 월요일):
  - 제목: `이번 주 바다상회에 맞는 공고 4건`
  - 본문: 공고 리스트 4행 (제목 / D-day / 금액 — 접힘 카드와 같은 절제) + 각 행 `자세히 보기`
  - 말미 전환 블록 (옅은 배경): `이번 주 마감 3건 — 실시간 알림이었다면 D-7에 아셨을 거예요` + 버튼 `멤버십 알아보기`
  - 푸터: 수신 거부 링크 (법적 필수)
- **마감 리마인더** (멤버십): 제목 `⏰ 부산 청년기업 스케일업, 마감까지 3일` + 본문: 공고 카드 1장 + 진행 상태 `서류 7/12 확인` + CTA `이어서 작성하기` (알림톡 채널 도입 여부는 미정 — 이메일 레이아웃 기준)

금지: 이메일에 3건 초과 CTA, 뉴스레터식 장문.
