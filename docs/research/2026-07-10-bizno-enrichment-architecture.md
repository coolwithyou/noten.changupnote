## 0. 요약 (TL;DR)

1. **팝빌 대체 판정: 부분 대체.** 휴폐업·과세유형·개인/법인 구분 3개 필드는 국세청 무료 API로 **지금 즉시 완전 대체**(이미 게이트로 사용 중 — 팝빌 과금 경로에서 제거만 하면 됨). **법인**은 금융위 기업기본정보 전량 벌크 수집으로 bzno→crno 인덱스를 자체 구축하면 상호·대표자·설립일·주소·중소기업여부·종업원수를 무료 확보하고, 덤으로 팝빌에 없던 **매출액·자산총액(금융위 재무 API)까지** 얻는다. **개인사업자의 상호·대표자·개업일·업태·종목은 어떤 공공 소스에도 없다**(국세청 비공개 정책) — 이 구간만 유료(팝빌 100원/건 유지 또는 NICE IF2150/LP0203 전환, 견적 비교 후 결정)로 남긴다. 유료 호출은 "무료 소스 전부 미스일 때만"으로 좁혀 호출률 자체를 낮추는 것이 구조의 핵심.
2. **NICE BizAPI는 전 API가 사업자번호 하나로 조회 가능**(companyKey 자동 판별). LP0203/OCOV02는 팝빌 상위호환 프로필(+종업원수·기업규모·KSIC 5단계·법인등기상태), OCOV06은 법인 5개년 재무, OCCD 계열은 **체납(신정원 공공정보)·법정관리** — 법인 신용정보 조회는 **법적으로 동의 불요**(법제처 해석). 단가는 전면 견적제.
3. **창업기업확인서 지분 이슈는 판정 트리로 사전화 가능.** 제외 사유의 정량 기준이 전부 확정됐고(50% 초과·과점주주·5천억 30% 최다출자·세세분류 동종), 2026-01 시행령 개정으로 지분형 사유(2·4·5호)는 **7년 내 해소 시 해소일부터 창업 인정**. 자동 판정 가능한 축(상호출자제한집단 대조, 폐업 후 3년 룰)과 설문 필요 축(친족 합산, 특수관계)을 분리한 pre-flight 모듈을 설계했다(§4).
4. 서류 발급의 3대 함정: 사실증명은 **대표자 본인 인증서로만** 온라인 발급, 주주명부는 공적 발급기관 없음(자체 작성+법인인감), **중소기업확인서 선발급이 크리티컬 패스**(창업기업확인 처리기한에 미포함). 공고 마감 역산 알림이 필요하다(§4.3).

## 2. 팝빌 대체 판정 — 필드×소스 매트릭스

팝빌 단가: 기업정보조회 **100원/건**, 휴폐업조회 10원/건 (기본료 없음, 선불).

| 팝빌 필드 | 법인 | 개인사업자 | 판정 |
|---|---|---|---|
| 휴폐업상태 / 과세유형 | 국세청 무료 | 국세청 무료 | ✅ **즉시 완전 대체** |
| 개인/법인 구분 | bizno 가운데 2자리 규칙(81·86~88=법인) + 교차 | 동일 | ✅ 완전 대체 |
| 상호 / 대표자명 / 주소 | 금융위 인덱스(무료) / DART | ❌ 통판·조달·고용사업장만 부분 | 법인 ✅ / 개인 ⚠️ |
| 개업일 | 금융위 fstOpegDt·설립일 (국세청 개업일과 상이 가능) | ❌ 없음 | 법인 △ / 개인 ❌ |
| KSIC | DART induty_code(공시법인) / 금융위 sicNm(분류명) | ❌ 정확 코드 없음 | ⚠️ 부분 |
| 업태 / 종목 | ❌ 공공 소스 부재 | ❌ | ❌ 유료만 |
| 기업규모코드 | 금융위 smenpYn(중소 여부만) | ❌ | ⚠️ 부분 |

**결론과 전략**
- **팝빌 휴폐업조회(10원)는 오늘 제거 가능** — 이미 국세청 게이트가 있으므로 캐시 갱신 경로만 국세청으로 통일.
- **법인 트랙**: 금융위 인덱스 히트 시 팝빌 불필요 + 재무까지 무료 획득. 미스(신설·무접점 법인)만 유료 폴백.
- **개인사업자 트랙**: 상호·대표자·개업일·업태·종목은 유료 소스가 유일. 선택지는 (a) 팝빌 유지(100원, 검증된 현행) (b) NICE IF2150/LP0203 전환(국세청 실시간·필드 더 풍부, 단가 견적 필요) (c) 자가신고+진위확인 무료 UX를 1차로 두고 유료는 보증 스탬프용. **권장: 무료 소스 → 자가신고+진위확인 → 유료 폴백의 3단 폭포**로 유료 호출률을 구조적으로 낮추고, NICE 견적이 팝빌 100원 대비 우위면 폴백 벤더만 교체.
- 팝빌 SDK 제거가 아니라 **폴백 격하**가 목표 — `serviceData.ts`의 "NTS 게이트 → 팝빌" 2단을 "NTS 게이트 → 무료 웨이브 → (미스 시) 유료 폴백"으로 재배선.

## 3. 인리치먼트 파이프라인 구조 (웨이브 설계)

기존 자산 재사용: `company_profiles`(EAV + source/confidence/as_of), `company_enrichment_cache`(provider별 원본), SMPP 커넥터 패턴(`packages/core/src/smpp/`), NTS 게이트(`serviceData.ts:445-456`).

```
사업자번호 입력
│
▼ Wave 0 · 게이트 (무료, <1s) ──────────────── [기존]
│  국세청 상태조회 → 미등록/폐업이면 중단(과금 차단)
│  bizno 규칙 → 개인/법인 분기
│
▼ Wave 1 · 무료 실시간 팬아웃 (병렬, ~2s)
│  공통: 근로복지공단(근로자수·성립일·사업장명·주소) + SMPP[기존]
│  개인: 공정위 통신판매(상호·대표자) + 나라장터
│  법인: (Wave 2 인덱스 로컬 조회 — 네트워크 불필요)
│
▼ Wave 2 · 벌크 인덱스 (사전 배치, 로컬 조회 0ms)
│  corp_registry_index: bzno→{crno, corp_code, 프로필 스냅샷}
│    금융위 기본정보 전량 + DART bizr_no 매핑 + KRX 상장
│  cert_index: 벤처·이노비즈·메인비즈·사회적기업 (상호 퍼지)
│  법인 히트 시: crno로 금융위 재무 lazy 호출(매출·자산)
│
▼ Wave 3 · 유료 폴백 + 심화 (조건부)
│  프로필 미완성 시: NICE LP0203/IF2150 (또는 팝빌 현행 유지)
│  법인 심화: OCOV06(5개년 재무), OCCD01→03/06(체납·법정관리),
│             OCST02(주주 — pre-flight 입력)
│
▼ Wave 4 · 사용자 동의/제출 (비동기, 확인서 pre-flight 연동)
│  자가신고+진위확인 / CODEF(사실증명·부가세증명) / 주주명부 업로드 OCR
│
▼ company_profiles upsert (dimension별 source·confidence·as_of)
   → 매칭 엔진 → next_question은 "남은 결손 축"만
```

### 3.1 구현 컴포넌트

| 컴포넌트 | 위치(제안) | 내용 |
|---|---|---|
| 커넥터 5종 (Wave 1) | `packages/core/src/{kcomwel,ftc-mailorder,g2b,nps}/` | SMPP 패턴 복제: fetch→정규화→positive-only 병합, 30일 캐시 |
| 벌크 인덱스 빌더 | `packages/core/src/registry-index/` + 배치 스크립트 | 금융위 전량 순회(쿼터 관리·재개 커서), DART corp_code 배치, 증분 갱신 크론 |
| `corp_registry_index` 테이블 | schema.ts 신규 | `bzno`(PK), `crno`, `dart_corp_code`, `profile_snapshot` jsonb, `sources` text[], `synced_at` |
| NICE 커넥터 | `packages/core/src/nicebiz/` | opengate 클라이언트(client-id/secret), LP0203·IF2150·OCOV06·OCCD01 우선. 스펙: `docs/research/nicebiz-api-specs/` |
| 유료 폴백 라우터 | `serviceData.ts` 확장 | "프로필 완성도 점수 < 임계치 && 무료 전부 미스"일 때만 유료 호출 (팝빌/NICE 벤더 스위치) |
| source enum 확장 | schema.ts `profile_source` | `kcomwel`, `ftc`, `g2b`, `nps`, `fsc`, `dart`, `nicebiz` 추가 (`codef`/`ocr`는 기존 예약) |

### 3.2 confidence 정책 (기존 팝빌 0.6~0.8 체계와 정합)

국세청 직결(상태·IF2150) 0.9 / 금융위·DART 인덱스 0.8 (synced_at 경과 시 감쇠) / 근로복지공단 근로자수 0.7 (성립일≠개업일 주의) / 통판·조달 0.65 / 상호 퍼지 매칭(벤처 등) 0.55 + 사용자 확인 시 승격 / 자가신고+진위확인 통과 0.85 / 자가신고 단독 0.5.

### 3.3 법적·계약 체크포인트

- 법인 신용정보(OCCD/OCGD) 조회는 동의 불요(법제처). **개인사업자 포함 시 약관에 조회 고지+동의 문구** 삽입 권장(금융위 해석상 개인정보 재분류 경계).
- NICE 계약서에 이용 목적을 "이용자 본인 기업 정보 조회 대행"으로 명시(재제공 제한 조항 대비).
- 금융위 주식발행정보 상업 이용 금지(§1.2). 국민연금 마스킹 데이터는 퍼지 매칭 결과를 단독 확정 근거로 쓰지 말 것.

## 4. 확인서 Pre-flight 모듈 — 지분 이슈 방비와 서류 체크

### 4.1 창업기업확인서 지분 판정 트리

법정 기준(전부 원문 확정, 상세 출처는 리서치 로그 참조):

```
[자동 판정 가능 — 시스템 데이터만으로]
├─ 업력: 사업개시일+7년(신산업 10년) 이내?          ← 팝빌/금융위 개업일
├─ 제외업종: 유흥주점·사행시설?                      ← KSIC
├─ 상호출자제한집단 소속?                            ← 공정위 명단(egroup) 대조
└─ 폐업 후 동종 재개: 대표자 폐업이력 + 3년(부도·파산 2년)
   + KSIC 세세분류(5단위) 동일?                      ← 사실증명(Wave 4) 확보 시 자동

[반자동 — 주주명부/OCST02 확보 시 자동, 미확보 시 설문]
├─ 자회사형(4호): 법인주주 1곳이 50% 초과?
├─ 5천억 나목: 자산 5천억↑ 법인이 30%↑ + 최다출자?   ← 법인주주 자산은 금융위 재무/DART
│   (VC·신기술금융·기술지주 지분은 산입 제외)
└─ 관계기업: 법인 간 직접 출자 30%↑ + 최다출자?
    ⚠️ "대표자 동일"만으로 관계기업 의제 금지 (법제처 유권해석 — 과잉판정 함정)

[설문 필수 — 데이터로 알 수 없음]
├─ 친족(배우자·6촌 혈족·4촌 인척) 합산 50% 초과 + 기존 개인사업 계속? (2호나목)
├─ 기존 법인 과점주주 구성과 신설법인 과점주주 겹침? (5호, 특수관계인 합산)
├─ 상속·증여·양수도 승계? (1호 — 해소 구제 불가)
└─ 개인→법인 전환 시 포괄양수도 여부 (6호)
```

**2026-01 개정 활용 (시행령 제2조 제2항 신설, 대통령령 35937호)**: 지분형 사유(2호나목·4호·5호)는 7년 내 해소하면 **해소일부터** 창업 인정 — ① 기존 사업 폐업일 ② 지분 50% 이하 하락일 ③ 과점주주 지위 상실일. **1호(승계)·3호(폐업 후 재개)·6호(조직변경)는 구제 대상 아님.** 해소 증빙은 주식등변동상황명세서·양수도계약서로 시점을 남겨야 함(주주명부는 등기사항이 아니라 변동이력 공부 부재). 형식적 명의 분산은 심사에서 부인 위험 — UI에 경고 필요.

### 4.2 사전 점검 질문지 (10문 — 설문 축)

1. 대표자가 현재 다른 사업자등록을 보유? (→ 사실증명으로 검증 가능)
2. 대표자(+배우자·6촌 혈족·4촌 인척)의 폐업 이력? 폐업일·업종은? (3년/2년 룰)
3. 대표자+친족 합산 지분 50% 초과이면서 기존 개인사업 계속 운영 중?
4. 다른 법인이 귀사 지분 50% 초과 보유?
5. 귀사 주주 구성이 기존 어느 법인의 과점주주 구성과 겹침?
6. 자산 5천억↑ 법인(외국법인 포함)이 직·간접 30%↑ + 최다출자자? (VC 제외)
7. 귀사가 타 법인 30%↑ 최다출자자거나, 그런 법인이 귀사 주주? (관계기업)
8. 사업을 상속·증여·양수도로 승계?
9. 개인→법인 전환이면 포괄양수도 계약 체결?
10. 사업개시일부터 7년(신산업 10년) 이내?

이슈 발견 시 안내: 해소 경로(지분 매각→50% 이하, 기존 법인 해산, 과점주주 지위 상실, 겸업 폐업)와 해소일 기산 규칙, 증빙 보존 안내.

### 4.3 서류 발급 체크리스트 (발급 가능성 사전 확인)

| 서류 | 발급처 | 소요 | 함정 (사전 경고 포인트) |
|---|---|---|---|
| 사업자등록증명 | 홈택스 무료 | 즉시~3h | 발급 3개월 이내분만 유효 |
| **사실증명(총사업자등록내역)** | 홈택스 무료 | ~3h, 15시 이후 익일 | **대표자 본인 인증서 필수** — 세무대리인·직원 발급 불가 |
| 법인등기부(제출용) | 인터넷등기소 1,000원 | 즉시 24h | 3개월 이내. 말소사항 포함 요구 여부 미확정(매뉴얼 재확인) |
| **주주명부** | **자체 작성** | — | 공적 발급기관 없음. 법인인감 날인·주민번호 마스킹·기준일 명시. 주식등변동상황명세서로 완전 대체 불가(변동 있던 해만 존재) |
| 폐업사실증명 | 홈택스 무료 | ~3h | 폐업신고 수리 후에만 발급 가능 |
| 주업종코드 확인자료 | 홈택스 | 즉시 | — |

**공통 반려 사유(형식)**: 원본대조필 누락, 주민번호 미마스킹, 화면 캡처본, 저해상도 스캔, 3개월 초과분.

**크리티컬 패스와 역산 알림**:
```
공고 마감일
 ◀ 창업기업확인: 서류 완비 접수 후 10영업일 (연장 시 20, 이의신청 포함 최장 ~47일)
 ◀ 중소기업확인서 선발급 필수: 자동이면 즉시, 관계기업 보유 시 수기심사 수일+
   (당해·직전연도 신설기업은 무서류 즉시 — 업력 1년 미만이면 지름길)
 ◀ 사실증명 등 서류 준비: 1~2일 (대표자 인증서 준비 포함)
→ 여유 없으면 "이 공고는 확인서 준비가 마감에 못 맞습니다" 경고 = 매칭 결과의 실효성 향상
```

소상공인확인서(동일 sminfo): 홈택스 인증서 연계 반자동 제출, 신설기업 무서류, 관계기업 시 오프라인. 유효기간 표준 2026-04-01~2027-03-31. 참고: 2025-09-01부터 중소기업 매출 상한 1,500억→1,800억.

### 4.4 매칭 엔진 연동

- 파생 판정 결과를 `company_profiles(dimension='certification', value={cert, status: eligible_unissued|held|ineligible|blocked_equity|unknown, blockers[], resolvable_by})`로 기록.
- `f_required_certs`에 확인서가 걸린 공고: `held`→추천 / `eligible_unissued`→"발급 절차 안내+마감 역산" 동반 추천 / `blocked_equity`→해소 경로 안내 / `ineligible`→제외.
- `next_question`을 pre-flight 설문 축(§4.2)과 통합 — 질문 1개당 판정 진전이 최대인 문항 우선.
