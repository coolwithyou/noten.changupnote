## 2. P0 — 오늘 IR 시연 중 화면에 바로 보이는 것

1. **`/pricing` 구 크레딧 모델** — 공개 URL이 "크레딧 플랜 선택 ×3 · 1크레딧=1원"을 노출. 피치("요금제는 하나예요, Free/멤버십 9,900원+이용권 5회 3,900원")와 정면 충돌.
   → 최소: 5a 프레임대로 정적 재작성(PricingView 교체, 데이터 fetch 불필요). 차선: 시연 동선에서 완전 회피.
2. **계정 메뉴 "플랜과 청구" → `/billing`(AI 크레딧 구독)** — 아바타 메뉴 한 클릭 거리에 크레딧 어휘 표면. `APP_ACCOUNT_LINKS`(app-navigation.ts)에서 임시 제거 또는 라벨 교체. (참고: app-sidebar의 "크레딧" 라벨은 사이드바 폐지로 비노출 — 무해)
3. **로그인 소셜 버튼 정본 충돌** — 6a 정본은 소셜 없음인데 `LoginPanel`이 env에 키 있는 OAuth(Google·Kakao)를 전부 노출. 시연 env에서 키 제거가 최속, 코드로는 `selectVisibleLoginMethods`에 플래그 1개.
4. **D-day 레드 임계 3원화** — NoticeCard ≤14 레드 vs 매칭 펼침 카드·신청 리스트 `isUrgentDday` ≤7. **같은 카드를 펼치는 순간 D-9가 레드→회색으로 변함.** `URGENT_MAX_DDAY`를 notice-card에서 export해 `match-results/logic.ts:791`·`ApplicationPipelineView.tsx:643`에 공유. 몇 줄 수정.
5. **사업자번호 마스킹 포맷 2원화** — 로딩 `123-**-*****`(클라) → 확인 `123-**-67***`(서버)가 연달아 다르게 보임. `biz-lookup-utils.ts:40-43` 1줄을 서버 `maskCorpNum` 포맷에 맞추면 끝.
6. **상태 C 게이지 캡션 자기모순** — 질문 소진+0건 화면에서 캡션이 여전히 "회사를 더 설명할수록 결과가 정확해져요". `ResultsHero`에 `questionsExhausted` prop 분기 1줄.
7. **캘린더 "내 캘린더에 연동" 메뉴** — 정본의 Google/Apple/Outlook 항목 대신 2항목이며, 구독 클릭 시 캘린더 앱이 아니라 **.md 파일이 다운로드**됨. 기존 webcal/https 구독 URL로 `webcal://`(Apple)·Google 추가 링크·Outlook 링크 3항목 확장. 시연에서는 클릭 회피도 가능.
8. **시연 운영 수칙** — 히어로 입력 포커스 시 최근 조회 제안(LookupSuggestions, 디자인에 없는 UI)이 localStorage 기록으로 뜸 → 시크릿 창 사용. `/credits`·`/account/usage` URL 진입 금지.
