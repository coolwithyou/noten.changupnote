# S0 스파이크 결과 — K-Startup 수집·정규화 PoC

실행일: 2026-06-24 · 표본 500건 / 전체 29,187건 · 스크립트 `poc/kstartup_normalizer_poc.py`

## 결론 한 줄
K-Startup은 **매칭의 척추(지역·업력·연령·분야·대상·일정)가 구조화 필드로 100% 제공**되어, **LLM 없이 파서만으로** 견고한 1차 매칭이 가능하다. LLM 추출은 짧은 scoped 텍스트의 *정밀 보강*에만 필요.

## 1. 구조화 커버리지 (표본 500)
| 차원 | 필드 | 채움률 |
|---|---|---|
| 지역 | supt_regin | 100% |
| 업력 | biz_enyy | 100% |
| 연령 | biz_trgt_age | 100% |
| 분야 | supt_biz_clsfc | 100% |
| 대상 | aply_trgt | 100% |
| 일정 | pbanc_rcpt_end_dt | 100% |
| 신청대상 상세 | aply_trgt_ctnt | 100% |
| 제외대상 | aply_excl_trgt_ctnt | 46% (해당 시) |

→ 척추 6개 차원은 **파싱만으로 자동 산출**(LLM 0, 오류 위험 0). 현재 open 215/500(누적엔 마감 다수 → 신선도 필터 필수 재확인).

## 2. 어휘(controlled vocabulary) 실측
- **지역**: 전국(56%) + 17개 시도. `전국→[](전역)`, 나머지 행정코드 매핑.
- **업력 biz_enyy**: {예비창업자, 1·2·3·5·7·10년미만} 콤마 집합 → `max_months`(상한) + `include_preliminary`. **예비창업 네이티브 포함.**
- **연령 biz_trgt_age**: {만20세미만, 20~39, 40+}. 3종 모두 = 제약없음, 부분집합 = 연령제한.
- **분야 supt_biz_clsfc**: 사업화/시설·공간·보육/멘토링·컨설팅·교육/행사·네트워크/판로·해외진출/창업교육/글로벌/R&D/정책자금/인력 (~10종).
- **대상 aply_trgt**: 일반기업/1인창조기업/일반인/대학생/대학/연구기관/청소년…

## 3. LLM 잔여 필요 (scoped 텍스트 한정, 휴리스틱 추정)
`aply_trgt_ctnt`+`aply_excl_trgt_ctnt`에서만:
| 보강 대상 | 언급 비율 |
|---|---|
| 업종·분야 상세 | 56.4% |
| 규모(중소/중견/소상공인/매출) | 28.2% |
| 인증(벤처/이노비즈/특허) | 17.2% |
| 지역 배제(수도권 제외 등) | 19.8% |

→ LLM 추출 대상은 **HWP·본문이 아니라 짧은 scoped 필드**이고, 전건이 아니라 *일부*. K-Startup 추출 위험·비용은 낮음.

## 5. 발견된 보정 포인트 (설계 반영 필요)
1. **지역 = supt_regin AND 제외텍스트 결합.** `전국` + `aply_excl_trgt_ctnt`의 "수도권 제외" = 실질 비수도권. 지역 판정은 두 소스를 합쳐야 정확(19.8%가 지역배제 보유). → region criteria 생성 시 제외텍스트 경량 추출 필수.
2. **biz_enyy 상한 해석**: 브래킷 집합의 최대값을 ceiling으로(예: 10년미만→120M). 예비 포함은 OR 조건.
3. **신선도**: 표본의 57%가 이미 마감 → 누적 29,187 중 active만 서빙. 상태 수명주기·전수정합 검증됨.

## 6. 엔드투엔드 매칭 데모 (설계 전체 실증) — `poc/kstartup_match_demo.py`
가상 사업자(**경기·업력 2년2개월·대표 35세**)를 open 215건과 매칭:

```
적격 63 · 조건부 88 · 부적격 64
[eligible 100] 스타트업 테크 브릿지   ✅지역:경기 ✅업력:7년이내·예비 ✅연령:35세
[eligible 100] 기보벤처캠프 19기      ✅지역:전국 ✅업력:3년이내 ✅연령:35세
[부적격] K-패션 글로벌 팝업    ❌ 서울 대상 — 귀사 경기
[부적격] 디지털 글로벌 역량강화 ❌ 수도권 제외 — 경기 해당      ← exclusion 동작
```
→ **설계한 매칭 엔진 전체가 실데이터에서 검증됨**: 필수/배제 평가, 적격·조건부·부적격 3분류, rule_trace 자연어 설명, 적합도 점수, 예비 OR 조건, 수도권 제외 배제까지. 조건부 88건은 업종/규모가 scoped 텍스트라 "확인 필요"로 정직하게 분류(= LLM 보강 대상).

## 7. S0-B — 기업마당 HWP 변환 스파이크 (✅ 해결)
실제 공고 첨부(`2026 SaaS 전환 컨설팅 모집공고.hwp`, **HWP 5.0 바이너리/OLE2, 74KB**) 변환 검증:

| 도구 | 본문 산문 | **표 내용** | 결론 |
|---|---|---|---|
| `hwp5txt` (pyhwp) | ✅ | ❌ `<표>`로 **통째 손실** | 부적합(자격요건이 표면 유실) |
| **`hwp5html` (pyhwp)** | ✅ | ✅ **11개 표 셀 단위 보존** | **채택** |

- hwp5html → 마크다운(본문+표) 변환 결과 4,732자, **모집대상·지원자격·선정절차·일정 표가 온전히 LLM-ready로 추출됨**. 샘플: `samples/bizinfo_hwp_converted.md`.
- **핵심**: HWP 표 손실 리스크는 **오픈소스 pyhwp `hwp5html`로 해결**(Rust/Docpler·markitdown-hwp 불필요). 변환 파이프라인은 `HWP → hwp5html → md(본문+표) → LLM 추출`.
- 잔여 케이스: **스캔 이미지형 HWP/PDF는 OCR 필요**(기존 fallback 유지). HWPX(신형)는 zip+XML이라 더 쉬움.

## 8. S0-C — LLM 추출 PoC (✅ 정성 검증, claude-sonnet-4-6) — `poc/llm_extract_poc.py`
실제 공고 5건(K-Startup scoped + 기업마당 HWP md)에 Claude 추출:

**품질(원문 대조 자체평가) = 높음.**
- 업력 7년 + "신산업은 10년 완화" 같은 **nuance 포착**(needs_review로).
- 배제업종(금융·보험·부동산·숙박)·**채무불이행/체납 배제**·OR 트랙 로직까지 추출.
- 자동판정 불가(입주 트랙 등)는 `text_only`+`needs_review`로 **정직하게 보류**(거짓확신 없음).
- 없는 자격은 **날조 안 함**(SaaS 공고처럼 hard 기준 없으면 비움).

**핵심 엔지니어링 교훈:**
1. **raw 텍스트 JSON 파싱은 불안정**(3건 중 1건 파싱 실패). → **tool-use(구조화 출력) 필수** — 전환하니 실패 0.
2. **operator/value 스키마 enum 강제 필요** — 같은 의미를 `lte`/`<=`/`==`로 흔들게 출력 → enum·dimension별 value 형태 고정.
3. **비용 기준**: 공고당 ~1.7k in / ~1.5k out 토큰(센트 단위). 증분+캐시면 backfill도 감당.

**한계(정직)**: n=5·사람 라벨 없음 → 이건 *작동·실패모드* 검증이지 P/R 숫자가 아님. **엄밀한 정확도는 도메인 전문가 골든셋(팀)** 필요. 단, 추출이 실무 수준으로 **작동함**은 실증됨.
