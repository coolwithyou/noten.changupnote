# 인터페이스 목업 디자인 브리프 — Claude Design용 (2026-07-14)

> **용도**: `2026-07-14-인터페이스-단순화-제안.md`의 새 인터페이스를 claude.ai/design 에서 페이지 단위 프로토타입으로 만들기 위한 브리프.
> **사용법**: **§0(공통 디자인 언어)을 항상 먼저 붙여넣고**, 이어서 원하는 페이지 섹션(P1~P8) 하나를 통째로 붙여넣어 페이지별로 생성한다. 샘플 데이터는 §0.4를 공유하므로 페이지 간 연결이 자연스럽다.

## §0. 공통 디자인 언어 (모든 페이지 생성 시 함께 붙여넣기)

### 0.1 제품과 사용자

- 제품명 **창업노트** — 사업자번호 하나로 받을 수 있는 공공 지원사업을 찾아주고 지원서 작성까지 안내하는 서비스
- 사용자: 소상공인·초기 창업자. 공고 문법(적격/우대/배제)에 익숙하지 않음. 바쁘고, 관공서식 화면에 지쳐 있음
- 디자인 철학: **"복잡한 엔진, 단순한 조종석"**. 화면은 결론과 다음 행동 하나만 크게 말하고, 근거·점수·상세는 사용자가 요청할 때만 펼친다. 토스의 "One thing per one page", TurboTax의 인터뷰 패턴이 레퍼런스

### 0.2 비주얼 토큰

- 배경 흰색, 본문 거의 검정(#222 수준), 보조 텍스트 회색(#6b7280 수준)
- **Primary 블루 #3182F6** (hover #1B64DA), 연한 블루 틴트 #E8F3FF
- **민트 #2BD4A8** (긍정·확정 신호), 민트 소프트 #E6FBF1
- 경고/마감임박 레드 #F04452
- 라디우스 크게: 카드 16px, 버튼 12px. 그림자는 아주 옅게. 여백 넉넉하게
- 타이포: 한글 산세리프(Pretendard 느낌). 헤드라인 굵고 크게, 본문은 15~16px. 숫자는 tabular
- 톤앤매너: 해요체, 짧은 문장. 관공서 용어 금지("귀하", "해당", "적격" 대신 "지금 신청 가능")

### 0.3 상태 어휘 (전 화면 공통 — 이 4+3개 외 상태 라벨 금지)

공고 판정 4상태 (뱃지):
| 문구 | 색 |
|---|---|
| **지금 신청 가능** | 민트 배경 + 진한 민트 텍스트 |
| **답하면 확정** | 블루 틴트 배경 + 블루 텍스트 |
| **원문 확인 필요** | 회색 배경 + 회색 텍스트 |
| **이번엔 어려움** | 연회색 배경 + 흐린 텍스트 |

지원서 필드 3상태: **채워짐**(민트 체크) / **확인만 하면 됨**(블루 점) / **입력 필요**(회색 빈 원)

보조 표기 2종 (뱃지가 아님 — 뱃지 어휘는 위 4개로 고정):
- **접수 예정** 공고는 D-day 자리에 `7/21 접수 시작`으로 표기
- **준비하면 열려요** 공고는 별도 뱃지 없이 결과 화면의 접힌 섹션 이름으로만 등장

금지 문구: "지원 가능성이 높음", "합격 확률" 등 확률 약속 표현 절대 금지. 점수(%)는 접힘 화면에 노출 금지.

- 회사: **주식회사 바다상회** (부산 · 음식료품 도소매 · 2023년 설립 · 대표 김서연) / 사업자번호 표기 `123-45-***90`
- 공고 샘플:
  1. **초기창업패키지** — 창업진흥원 · D-12 · 최대 1억 원 · 지금 신청 가능
  2. **청년창업사관학교** — 중소벤처기업진흥공단 · D-30 · 최대 1억 원 · 지금 신청 가능
  3. **소상공인 성장지원 사업** — 소상공인시장진흥공단 · 상시 · 최대 2,000만 원 · 지금 신청 가능
  4. **수출바우처 (내수기업 트랙)** — KOTRA · D-21 · 최대 3,000만 원 · 답하면 확정
  5. **부산 청년기업 스케일업** — 부산경제진흥원 · D-9 · 최대 5,000만 원 · 답하면 확정
  6. **스마트상점 기술보급** — 소상공인시장진흥공단 · D-45 · 최대 500만 원 · 원문 확인 필요
  7. **예비창업패키지 2차** — 창업진흥원 · `7/21 접수 시작` · 최대 1억 원 · 접수 예정(보조 표기)
  8. **이노비즈 연계 기술개발** — 중소벤처기업부 · 상시 · 최대 5,000만 원 · 준비하면 열려요 (열림 조건: 이노비즈 인증 취득)
- 작성 지원 방식(공고별로 다름 — P5·P6에서 사용): 공고 4 = **원본 양식 채움**(HWP 양식을 자동으로 채워 HWPX로 내려받음) / 공고 3 = **초안 작성**(원본 양식이 없어 AI 초안으로 도움) / 공고 6 = **웹폼 안내**(온라인 접수라 항목 안내만)
- 대표 질문: **"2025년 연 매출이 10억 원 미만인가요?"** → 보상 문구 "답하면 11건 중 최대 9건이 확정돼요"
- 지원서 필드 샘플: 상호명(자동: 주식회사 바다상회) / 사업자등록번호(자동) / 대표자명(자동) / 창업 아이템 한 줄 소개(입력 필요) / 최근 연 매출(확인 필요: 제안 3.2억 원)

**필드 목록의 근거 (실코드 대조 완료 — 임의 필드 추가 금지)**
- P4 시트의 11개 필드는 실제 UI 계약(`buildCompanyEvidenceFields`, 2026-07-14 대조)과 1:1 동일: 상호 / 소재지 / 업력 / 기업규모 / 업종 / 영업상태 / 사업자 유형 (이상 자동 확인 7) + 대표자 연령 / 보유 인증·확인서 / 상시근로자 / 연 매출 (이상 직접 입력 4)
- 매칭 알고리즘은 22축(`CRITERION_DIMENSIONS`)을 지원하며, 위 11필드가 커버하지 않는 축(결격 3축: 세금 체납·신용·제재 / 재무·고용·투자 수치 / 기존 수혜 이력 / 대표자 특성 / 지식재산권)은 **P3의 질문 카드가 담당**한다(결격 체크리스트·수치 묶음·수혜 이력 폼). 목업에서 이 축들을 P4 시트에 그리지 말 것 — 질문 카드의 소재로만 사용
- `수출 실적`·`사업장 보유`는 분류 **내** 축(승격 대기)이므로 "분류에 없는 조건"의 예시로 쓰지 말 것

### 0.5 화면 규격

- 데스크톱 1440px 기준. P3(매칭 결과)·P6(워크스페이스)은 모바일 390px 변형도 함께
- 상단 헤더 공통: 로고 `창업노트`(블루 C 심볼) 좌측 / 우측 `로그인` 버튼 1개 (로그인 후엔 아바타). 내비게이션 링크는 최대 3개 이하

## 전체 여정 (프로토타입 연결 순서)

```
P1 랜딩 → P2 회사 확인 다이얼로그 → P3 매칭 결과(정밀도 루프) ⇄ P4 내 정보 시트
                                        │
                                        ▼
                              P5 공고 요약 → (지원서 작성 화면은 이번 라운드 범위 외)
                                        │
                                        ▼
                              P7 신청 관리 · P8 대시보드(재방문 홈)
```

> **이번 라운드 범위**: 순수 사용자용 화면만.
> **P6 범위 재결정(2026-07-14 2차)**: 처음에 "워크스페이스=리뷰어용"으로 보고 제외했으나, 이는 내부 검수용 리뷰어 워크스페이스(internal/review)와의 혼동이었음. P6(지원서 작성 도우미 — HWP 렌더링·칸별 설명·값 추천/삽입)는 **사용자용 핵심 기능이므로 범위 복귀** — 전달 프롬프트의 화면 8. P6b(초안·웹폼 모드)는 후속 라운드로 보류.

### 이번 라운드의 디자인 포인트 (P3·P4 관통 원칙)

1. **자동 vs 직접의 명확한 구분** — 사업자번호로 자동 확인된 정보와, 고객이 직접 채워야 하는 정보가 한눈에 구분되어야 한다 (자동=민트 체크, 직접=블루 빈 슬롯)
2. **채우고 싶게** — 빈 항목은 "부족함"이 아니라 "채우면 열리는 보상"으로 보여야 한다. 모든 빈 슬롯 옆에는 채웠을 때의 보상이 미리 표시된다 (`+정밀도`, `공고 N건 판정에 영향`)
3. **채울 때마다 정밀도가 오른다** — 화면 상단의 **매칭 정밀도 게이지**가 답변·입력 때마다 눈에 띄게 상승
4. **채울 때마다 결과가 좁혀진다** — 확정되는 공고(민트로 이동)와 제외되는 공고(흐려지며 빠짐)가 실시간으로 보인다. 단, 답변이 결과를 바꾸지 않는 경우도 있음을 자연스럽게 처리 (`이 답변으로 바뀐 공고는 없었어요 — 정밀도는 올라갔어요`)
5. **분류 밖 조건은 묻지 않고 조언한다** — 우리 분류(22축)에 없는 공고 고유의 세부 조건은 질문·채팅으로 수집하지 않고, 카드 안에서 **AI 조언 문장**으로 안내한다 ("이 공고는 ~가 해당되어야 지원할 수 있어요"). 좁혀짐(원칙 4)은 분류 내 질문 카드가 담당

## P1. 랜딩

**제작 지시**: 스크롤 5섹션의 초단순 랜딩. 이 페이지의 목표는 단 하나 — 사업자번호 입력. 화면 자체가 "30초면 끝난다"는 인상을 줘야 한다.

**섹션 구성 (위→아래, 이 5개가 전부. 추가 섹션 금지)**

1. **히어로** (첫 뷰포트의 주인공)
   - 상단 작은 뱃지: `● 지금 신청 가능한 지원사업 1,284건` (초록 점)
   - 헤드라인 2줄: `사업자번호만 넣으면,` / `받을 수 있는 지원사업이 보여요` (두 번째 줄 일부 블루 그라디언트 강조)
   - 대형 입력 폼 1개: placeholder `000-00-00000` + 블루 버튼 `지원사업 찾기`
   - 폼 아래 캡션 한 줄(회색, 작게): `회원가입 없이 30초 · 입력 정보는 암호화돼요`
   - 이 외 요소 금지 (칩 나열, 보조 CTA 없음)
2. **실공고 프리뷰 밴드**: 캡션 `매주 40여 개 기관의 공고를 수집해요` + 카드 3장 가로 배치. 카드는 §0.4의 공고 1·4·5를 사용하되 **카드당 3요소만**: 제목 / D-day 뱃지 / `최대 1억 원`
3. **3단계 안내**: 제목 `공부 없이, 세 단계면 충분해요` + 번호 카드 3장 — ① `사업자번호 입력` "번호 하나면 회사 정보를 자동으로 불러와요" ② `맞춤 매칭` "지금 신청 가능한 사업과 답하면 확정되는 사업을 나눠 보여줘요" ③ `신청 코칭` "서류의 80%는 미리 채워드리고, 빠진 것만 물어봐요"
4. **마지막 CTA**: 헤드라인 `사업자번호 하나로 지금 바로 시작하세요` + 히어로와 동일한 입력 폼
5. **FAQ + 푸터**: 아코디언 4개(기본 모두 닫힘) — "정말 사업자번호만 넣으면 되나요?" / "회원가입을 꼭 해야 하나요?" / "어떤 지원사업을 다루나요?" / "비용이 있나요?". 푸터는 한 줄(© 2026 바톤 + 이용약관 · 개인정보처리방침)

**금지**: 기능 나열 섹션, 통계 숫자 섹션, 아이콘 그리드, 두 개 이상의 서로 다른 CTA 문구.

## P2. 회사 확인 다이얼로그 (P1 위 오버레이)

**제작 지시**: 랜딩 위에 뜨는 확인 모달 1장. 3상태를 각각 목업.

1. **로딩**: 스피너 + `사업자 정보를 확인하고 있어요` + `123-45-***90`
2. **확인** (메인): 타이틀 `『주식회사 바다상회』 회사가 맞으신가요?` + 보조 정보 한 줄 `123-45-***90 · 영업 중 · 부산` (뱃지 3개가 아니라 중점(·) 구분 텍스트 한 줄) + 버튼 2개: 블루 `네, 매칭 결과 보기` / 고스트 `아니요, 다시 입력할게요`
3. **오류**: `등록된 사업자를 찾지 못했어요` + 안내 한 줄 + 버튼 `사업자번호 다시 입력`

## P3. 매칭 결과 — 정밀도 루프 화면 (★ 핵심, 가장 공들일 것)

**제작 지시**: "공고 목록"이 아니라 **"회사를 설명할수록 정밀도가 오르고 결과가 좁혀지는 루프"**로 보여야 한다. 자동으로 확인된 것과 직접 채울 것의 구분이 명확하고, 빈 곳을 채우고 싶어지게 만드는 것이 이 화면의 성패. 상태 변형 3종(A/B/C)을 모두 목업. 데스크톱 + 모바일.

**레이아웃 (위→아래, 단일 칼럼 max-width 760px)**

1. **결과 헤드라인** (한 문장, 큰 타이포): `지금 신청 가능 3건 · 답하면 확정 11건` — 숫자 두 개만 강조(민트/블루). 아래 작은 캡션: `7월 14일 기준, 공고 1,284건과 대조했어요`
   - 우측 상단 보조 버튼 1개: `결과 저장하기`
2. **매칭 정밀도 게이지** (헤드라인 바로 아래, 얇은 한 줄 — 이 화면의 동기 장치)
   - `매칭 정밀도 64%` + 가로 게이지 바(민트 채움) + 캡션 `회사를 더 설명할수록 결과가 정확해져요`

## P7. 신청 관리

**제작 지시**: 칸반 보드가 아니라 **단순 리스트**. 신청 건수가 3~5건일 때 자연스러워 보여야 한다.

- 헤더: `내 신청 현황` + 보조 버튼 1개 `캘린더로 보기`
- 그룹 3개(리스트 구분 헤더): **진행 중 (2)** / **결과 대기 (1)** / **종료 (1)**
- 행 구성 (한 줄): 공고명 / 상태 문구 한 줄(예: `서류 7/12 확인 · D-21`) / 주 행동 버튼 1개 (`이어서 작성` 또는 `결과 입력`) / ⋯ 메뉴
  - 진행 중: `수출바우처 (내수기업 트랙)` — `서류 7/12 확인 · D-21` — `[이어서 작성]`
  - 진행 중: `초기창업패키지` — `서류 확인 전 · D-12` — `[작성 시작]`
  - 결과 대기: `소상공인 성장지원 사업` — `6월 30일 제출함` — `[결과 입력]`
  - 종료: `2025 창업도약패키지` — `선정 🎉` — (버튼 없음, 흐리게)
- ⋯ 메뉴 하나 열린 모습: `메모` / `리마인더` / `보류로 이동`

**금지**: 8개 통계 카드, 8개 토글, 8레인 보드, 헤더 링크 5개.

## P8. 대시보드 (재방문 홈)

**제작 지시**: 로그인 사용자가 매일 아침 여는 화면. 구조는 **"다음 행동 카드 1개 + P3 재사용"** — 새로운 UI를 발명하지 않는다.

- 인사 한 줄: `바다상회님, 오늘 확인할 것 하나예요`
- **다음 행동 카드 1장** (P3의 질문 카드와 같은 시각 문법, 블루 틴트): 
  - 예시: `⏰ 부산 청년기업 스케일업이 9일 뒤 마감돼요` + 부연 `질문 1개에 답하면 신청 가능 여부가 확정돼요` + 주 CTA `지금 확인하기`
- 그 아래는 **P3와 동일한 결과 화면** (상태 탭 3개: `지금 가능 9` / `답하면 확정 5` / `전체`) — P3 컴포넌트 재사용임을 목업에서도 동일한 모양으로 표현
- 우측 상단 어딘가에 작은 링크 2개만: `내 신청 현황` (→P7) / `내 정보` (→P4)

**금지**: 기회 맵 4버킷 보드, 로드맵 스트립, 할 일 큐 패널 — 이 셋은 "다음 행동 카드 1장"으로 대체된 것.
