# CODEF 제출용 — 창업노트 이용 상품군 목록

> 2026-07-13 작성. CODEF 정식 전환·견적 문의에 첨부할 상품 목록.
> 전제: 현재 코드베이스의 외부 데이터 API(팝빌·공공데이터포털 국세청·근로복지공단·Apick·NICE)를
> CODEF로 최대한 통합하는 시나리오. 상품 경로는 developer.codef.io 개발가이드 기준.

## 서비스 개요 (제출 시 함께 전달)

- 서비스: 창업노트(changupnote.com) — 사업자번호 기반 공공 지원사업 매칭·지원서 작성 안내 SaaS
- 이용 목적: 이용자 본인 기업 정보 조회 대행 (이용자가 간편인증으로 동의 후 조회, 재제공·재판매 없음)
- 호출 패턴: 신규 기업 온보딩 시 증명류 각 1회 조회 후 자사 캐시, 상태조회는 보유 기업 대상 주기 갱신
- 물량 시나리오: 월 신규 온보딩 1,000 / 10,000 / 50,000건 3구간 견적 요청

## 핵심 확인 사항 (견적 회신 시 요청)

1. A군 2건을 같은 간편인증 세션으로 연속 호출할 때 과금 단위 (인증 1회 기준인지 상품별인지)
2. 간편인증 미완료·타임아웃·빈 응답(무신고 사업자 등) 건의 과금 여부
3. 조회 결과의 자사 DB 저장·캐싱 허용 범위 (재조회 과금 최소화 목적)
4. B-1/B-2/B-3의 역할 구분과 권장 조합 (실시간 1건 vs 배치)
5. 상품별 단가(3구간)·월 기본료·최소 약정, 정식 전환 심사 기간
