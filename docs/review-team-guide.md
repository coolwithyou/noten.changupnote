# 검수팀 가이드 — 필드맵 라벨 검수

> 대상: 비개발자 리뷰어. 화면만 있으면 됩니다 (설치·코드 불필요).
> 이 가이드의 절차는 2026-07-03 실제 검수 시뮬레이션(저장→확정→취소 왕복)으로 검증되었습니다.

## 0. 이 작업을 왜 하나요 — 그리고 무엇을 얻나요

창업노트는 공고 첨부서류(HWP/PDF)에서 "지원자가 채워야 할 칸"(필드)을 자동으로
찾아내는 시스템을 만들고 있습니다. AI가 45개 서류에서 1,579개 필드를 미리 찾아놨지만,
**AI가 스스로 만든 답을 AI 채점 기준으로 쓸 수는 없습니다.** 사람이 확정한 정답지(golden set)가
있어야 비로소:

1. **측정이 가능해집니다** — 여러 자동 추출 엔진 후보를 "정답 대비 몇 %를 찾았나"로
   숫자 비교해 선택할 수 있습니다 (커버리지 80% 이상이 통과 기준)
2. **안전장치가 검증됩니다** — 서명·동의·직인처럼 반드시 사람이 직접 써야 하는 항목을
   시스템이 99% 이상 잡아내는지 확인합니다. 이게 뚫리면 사용자가 접수 거절을 당합니다
3. **시스템이 계속 배웁니다** — 여러분의 확정 하나하나가 일회성 검사가 아니라
   시스템이 앞으로 모든 서류를 더 정확히 읽게 만드는 영구 자산으로 쌓입니다

여러분이 확정 버튼을 누르는 순간 그 문서는 "정답"으로 승격되고, 시스템 채점에 바로 쓰입니다.
**45개 문서 검수가 끝나야 다음 개발 단계가 시작됩니다** — 지금 이 작업이 전체 일정의 관문입니다.

## 1. 접속

1. `dev.changupnote.com` 에서 **구글 로그인** (등록된 이메일이어야 합니다 — 안 되면 관리자에게 이메일 등록 요청)
2. 주소창에 `dev.changupnote.com/internal/review` 입력
3. 문서 45건 목록이 보이면 준비 완료. ("페이지를 찾을 수 없음"이 뜨면 등록되지 않은 계정입니다)

## 2. 화면 구성

**목록 화면**: 문서별 상태(대기/검수 중/확정)·필드 수·진행률. **교정 노트 뱃지**가 붙은
문서는 미리 알려진 주의사항이 있는 문서입니다 (10건).

**검수 화면** (문서 클릭):

- 왼쪽: 서류 페이지 이미지. 필드를 클릭하면 해당 위치에 **빨간 상자(bbox)** 강조.
  **이미지의 상자를 직접 클릭해도** 오른쪽 목록에서 해당 필드로 이동합니다
- 오른쪽: 필드 목록. 이름(key), 라벨, 종류, 체크 3종(아래 4장 참조), 메모(notes) 수정 가능.
  **검색창**으로 필드 이름·라벨을 바로 찾을 수 있습니다
- 오른쪽 상단 **"확인 필요만" 버튼**: AI가 스스로 자신 없다고 표시한 필드만 전 페이지에서
  모아 봅니다. 필드를 클릭하면 해당 페이지로 자동 이동합니다
- **상자 다시 그리기**: 필드를 선택한 뒤 "bbox 다시 그리기"를 켜고 이미지 위에서
  마우스로 드래그하면 상자가 새로 그려집니다 (Esc로 취소)
- 상단: 미리 알려진 교정 노트(있는 경우) + **화면 안 가이드**(`/internal/review/guide`) 링크

체크 3종의 뜻 (화면에도 표시됩니다):

- **필수 (required)**: 양식에 별표·"필수" 표기가 있거나, 안 쓰면 접수가 안 되는 항목
- **지원자가 작성 (applicantFills)**: 지원자가 채우는 칸 (담당 공무원 기재란이면 해제)
- **자필·서명 필요 (manual)**: 서명·도장·자필 동의처럼 반드시 사람이 직접 해야 하는 항목 —
  **가장 중요한 체크입니다**

## 3. 문서 1건 검수 절차 (10~20분)

우선순위 순서대로 봅니다. **누락 필드 찾기 > 잘못된 분류 고치기 > 상자 위치**입니다.

1. **"확인 필요만" 먼저** — AI가 자신 없어 한 필드부터 판정하고 notes의 "확인 필요" 문구를
   지우거나 결론으로 바꿉니다
2. **누락 찾기** — 페이지를 넘기며 "지원자가 써야 하는데 목록에 없는 칸"을 찾아
   [+ 필드 추가]. 특히 **말미 서명행**(신청인 ___ (인))은 자주 누락됩니다
3. **분류 확인** — 아래 3개가 가장 중요합니다:
   - **manual**: 서명·직인·자필 동의 등 "사람이 반드시 직접" 항목이 manual=true인가
     (생년월일 단독란은 manual 아님 / 주민번호 기입란은 manual 유지)
   - **type**: 체크박스인데 text로 되어 있지 않은가, 표(table)인데 낱개 필드로 쪼개져 있지 않은가
   - **required**: 별표·"필수" 표기가 있는 항목만 true인가
4. **상자 위치** — 크게 어긋난 것만 "bbox 다시 그리기"로 고치면 됩니다.
   미세 조정은 불필요 — "어느 칸을 가리키는지 구분되는 수준"이면 충분하고,
   정밀 좌표는 다음 단계에서 기계가 재계산합니다
5. **[저장]** — 중간 저장. 상태가 "검수 중"으로 바뀝니다
6. **[검수 확정]** — 이 문서를 정답으로 승격. **실수해도 [확정 취소]로 되돌릴 수 있습니다**

## 4. 판정이 헷갈릴 때

- 기준서(`docs/gate1-field-map-labeling-guide.md`)의 규칙 1~10과 표준 key 사전이 원칙입니다.
  key를 새로 만들기 전에 사전에 같은 의미가 있는지 확인하세요 (기업명은 항상 `company_name`)
- 그래도 애매하면 필드의 **[보류] 토글**을 켜고 사유를 적은 뒤 저장하세요 (확정하지 말 것).
  보류 수는 목록 화면에 표시되고, 운영자가 모아서 기준서 "판정 사례집"에 결론을 등재한 뒤
  알려드립니다. 문서 전체에 대한 질문·전달 사항은 **문서 코멘트**란에 적으세요
- 원본 서류를 직접 열어봐야 하는 문서(doc28 등)는 교정 노트에 표시되어 있습니다 —
  해당 건은 운영자에게 원본 파일을 요청하세요

## 5. 작업 순서 (권장)

1. 교정 노트 뱃지가 있는 10건 중 **소급 교정 대상 (doc05, doc10, doc23, doc29)** 먼저
2. 나머지는 목록 순서대로. 하루 4~6건이면 2주 내 완료됩니다
3. 같은 계열 문서(비슷한 양식)는 몰아서 하면 key 통일이 쉬워집니다

## 6. 하지 말아야 할 것

- **추측으로 확정하지 않기** — 확정은 "이 문서의 모든 필드에 책임진다"는 뜻입니다.
  불확실하면 저장 + 판정 보류 메모
- 이미지에 없는 내용을 상상으로 추가하지 않기 (원본 확인이 필요하면 요청)
- 필드 삭제는 신중히 — "중복이라서"가 아니라면 삭제 대신 notes에 이유를 남기고 보류
