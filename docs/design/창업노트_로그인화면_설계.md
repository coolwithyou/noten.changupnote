# 창업노트 — 로그인 화면 구성 계획

최종 갱신: 2026-06-27 · 상태: **1차 구현 완료(이메일/패스워드 + Google + Kakao), 랜딩 연결 완료**

NextAuth(JWT 세션) + DrizzleAdapter(`CUNOTE_AUTH_DB_ADAPTER=drizzle`) 기반. 회사별 기회 맵·신청 준비 상태를 저장하기 위한 계정 게이트.

---

## 1. 진입점 (랜딩 ↔ 로그인 연결)

| 진입 경로 | 동작 | 상태 |
|---|---|---|
| 랜딩 헤더 **`로그인` CTA** | `/login`으로 이동 (`HomeExperience` 헤더) | ✅ 연결됨 |
| 대시보드 접근 시도(미인증) | `redirectToLoginForDashboard()` → `/login?callbackUrl=/dashboard` | ✅ 기존 동작 |
| 로그인 완료 | `callbackUrl`(기본 `/dashboard`)로 복귀 | ✅ |
| 이미 로그인 상태로 `/login` 진입 | 서버에서 세션 감지 → `callbackUrl`로 redirect | ✅ (`login/page.tsx`) |

`callbackUrl`은 `normalizeCallbackUrl()`로 검증(내부 경로만 허용, `//` 차단)하여 오픈 리다이렉트를 방지한다.

## 2. 인증 수단 (현재)

| 수단 | provider id | 활성 조건 | 상태 |
|---|---|---|---|
| 이메일/비밀번호 | `password` | 항상 | ✅ 구현 |
| Google | `google` | `GOOGLE_CLIENT_ID`+`SECRET` 존재 | ✅ 키 설정·활성 |
| Kakao | `kakao` | `KAKAO_CLIENT_ID`+`SECRET` 존재 | ✅ 키 설정·활성 |
| 데모 로그인 | `demo` | `CUNOTE_AUTH_MODE=mock` (UI 노출은 mock만) | 개발 전용 |

- OAuth 리다이렉트 URI 기준 = `https://changupnote.com/api/auth/callback/{google|kakao}` (NEXTAUTH_URL과 일치).
- provider 노출은 서버 `getWebAuthProviderSummaries()`가 결정 → 클라이언트로 전달.

## 3. 화면 상태(state machine)

`LoginPanel`(클라이언트 컴포넌트)이 단일 카드 안에서 다음 상태를 전환한다.

```
login(기본) ⇄ register   (토글 링크)
  │              │
  ├ 제출 ─ pending ─ 성공 → window.location(callbackUrl)
  │                   └ 실패 → error 표시(폼 유지)
  └ OAuth 버튼 → signIn(provider) → 외부 동의 → callback
```

| 상태 | UI |
|---|---|
| login | 이메일·비밀번호 입력 + `로그인` + 회원가입 토글 + (divider) + OAuth 버튼 |
| register | 이름(선택)·이메일·비밀번호 + `가입하고 시작` + 로그인 토글 |
| pending | 해당 버튼 Spinner + 전체 입력 disabled |
| error | `Alert(destructive)` (aria-live=polite) |
| 수단 없음 | "활성화된 로그인 수단이 없습니다" 안내 |

비밀번호 정책: 8자 이상(`validateCredentials`), 이메일 형식 검증, 최대 200자.

## 4. 컴포넌트 구성

```
app/login/page.tsx (server)
  ├─ getOptionalWebSession() → 로그인 상태면 redirect
  ├─ getWebAuthProviderSummaries() → providers
  └─ <LoginPanel callbackUrl providers />   (client)
        ├─ 이메일/비밀번호 <form> (shadcn Input/Label/Button/Spinner)
        ├─ Separator("또는")
        ├─ OAuth <Button variant=outline>  (google/kakao)
        └─ Alert(error) · "처음 화면으로" 링크
```

디자인 토큰/컴포넌트는 shadcn/ui + Toss 디자인 시스템(`DESIGN.md`)을 따른다. 구조 클래스(`login-shell`/`login-panel`/`login-provider-list`)는 기존 CSS 유지, 신규 요소(divider·토글)는 Tailwind 유틸리티.

## 5. 플로우 & 계약(contracts)

### 회원가입
`POST /api/web/auth/register` `{ email, password, name? }`
- 검증 실패 → 400, 중복 이메일 → 409, 성공 → 201 `{ id, email }`
- 비밀번호는 bcrypt(rounds 12) 해시로 `users.password_hash`에 저장(평문 미저장).
- 가입 직후 클라이언트가 동일 자격으로 `signIn("password")` 자동 호출 → 세션.

### 로그인(credentials)
NextAuth `CredentialsProvider(password).authorize` → `users` 조회 → `verifyPassword` → 통과 시 `{ id, email, name }` 반환 → JWT(`sub=user.id`).

### 세션
`session` 콜백이 `token.sub`를 `session.user.id`로 부착 → 회사 권한/RLS(`app.current_user_id`)의 주체로 사용.

### OAuth
`signIn(google|kakao)` → 외부 동의 → `/api/auth/callback/*` → DrizzleAdapter가 `users`/`accounts` upsert.

---

## 6. 보안

- 비밀번호: bcrypt rounds 12, 서버에서만 해시/검증, 응답에 해시 미노출.
- CSRF: NextAuth csrfToken(쿠키+바디) 검증.
- `users`는 RLS 비대상(adapter가 직접 관리), PII 테이블은 RLS+`app.current_user_id`.
- 오답 비밀번호/미가입/소셜전용(해시 없음) 계정은 동일하게 `null` 반환(계정 존재 노출 최소화).
- 오픈 리다이렉트 방지(`normalizeCallbackUrl`).
