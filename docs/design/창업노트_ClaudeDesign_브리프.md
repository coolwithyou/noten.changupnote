# 창업노트 — Claude Design 브리프 & 프롬프트

목적: 아래를 그대로 Claude에 붙여 인터랙티브 UI 프로토타입(HTML/React)을 생성. 자기완결적.

---

## ▶ 붙여넣는 프롬프트 (요약본)

> 한국 스타트업·중소기업을 위한 **정부 지원사업 매칭 서비스 "창업노트"**의 인터랙티브 프로토타입을 만들어줘. 사업자번호 하나로 *지원 가능한 사업*을 찾아주고, *언제까지 뭘 해야 하는지*와 *지금 당장 뭘 해야 하는지*까지 안내하는 서비스야.
>
> **핵심 컨셉 — "지루한 공공기관 목록"이 아니라 RPG 게임처럼.** 지원사업 = *퀘스트*(조건 충족 시 활성화), 내 회사 = *성장하는 캐릭터*(업력·인증·투자단계로 레벨업), 서비스 = *퀘스트를 안내하는 NPC*("지금 이거부터 하세요"). 단, **게임처럼 보이되 모든 숫자·상태는 진짜 자격 판정에서 계산된 사실**이어야 해(과장·가짜 포인트 금지).
>
> **만들 화면 4개**: ①첫 화면(큰 숫자 훅 + 사업자번호 입력 + 개인화 카운트업) ②기회 맵(퀘스트 카드 보드: 적격/조건부/잠금 상태) ③다음 할 일(NPC 액션 큐: 레버리지순 행동 리스트) ④신청 준비 시트(조건 체크리스트 + 서류 + 신청 버튼). 보조로 ⑤성장 트랙(투자단계 예비→Seed→A→B→C + 업적).
>
> **톤**: 깔끔하고 세련된 토스(Toss)급 핀테크 톤. 미니멀·명료·신뢰가 우선. 게임 요소(퀘스트·상태·성장·액션)는 *정보 구조와 상태*로 표현하되 비주얼은 절제(과한 장식·네온·만화풍 금지). 한국어 UI, 라이트+다크 모드, 모바일 우선.
>
> 아래 "상세 브리프"의 화면 명세·상태 규칙·실제 샘플 데이터·금지사항을 따라 만들어줘.

## 1. 제품 한 줄 + 사용자
- 사업자번호 → 지원 가능한 정부지원사업 자동 매칭 + 일정 + 액션플랜.
- 사용자: 자금 부족하거나 투자받고 싶은 (예비)창업자·중소기업 대표. 진지한 의사결정자.
- 3대 니즈 = ①뭘 지원 가능한지 ②언제까지 뭘 ③지금 당장 뭘.

## 2. 핵심 차별화 (3축)
1. **토스식 첫 화면 훅** — 큰 숫자 하나 + 단일 입력 + 즉각 개인화. (회원가입 먼저·장황한 설명 금지)
2. **목록 탈피** — 표/리스트+필터 대신 *퀘스트 카드 보드 + 2D 로드맵*.
3. **게임화 월드** — 지원사업=퀘스트, 회사=성장 RPG 캐릭터, 서비스=NPC(액션 가이드).

## 3. 신뢰 가드레일 (불가침 — 디자인에 반영)
- **확정 vs 추정 분리**: 적격 카드만 "빛나는" 상태. 조건부는 *눈에 띄게 미확정*(흐림/점선). 금액은 "최대 ₩X (적격 N건 기준)"처럼 근거와 함께.
- **모르면 "확인 필요"** — 절대 거짓확신·과장 금지. 자동 판정 불가 조건은 "공고 원문 확인"으로 표기.
- 부적격도 숨기지 말고 *사유와 함께* 접어두기(펼침 가능).
- 가짜 포인트·전체 사용자 랭킹 금지(사업자끼리 민감). 보상은 *진짜*(해금된 사업·지원금·성장)에만.

## 4. 비주얼 언어
- **▶ 노스스타 레퍼런스 = `창업노트_카드_레퍼런스.html`** (이 톤·레이아웃을 기준으로). 다크 우선·세련·명료.
- **히어로 컴포넌트 = "표준화 지원사업 카드 + 조건별 판정"**. 모든 사업을 *같은 축*(지역·업력·업종·규모…)으로 조명하고, 조건마다 ✓충족 / ?확인 필요 / 🔒미해당을 명시 → 이게 제품 차별화의 핵심(경쟁사의 날것 목록 vs 우리의 표준화 판정). `grant_criteria`·`rule_trace`의 직접 시각화. 템플릿이라 손그림 없이 1,500+ 확장.
- **총괄 톤 = 깔끔하고 세련된 톤(다크 우선·라이트 지원)** — 미니멀·명료·신뢰. 게임 요소(퀘스트·상태·성장·액션)는 *정보 구조와 상태*로 표현, 비주얼 스킨은 절제(과한 장식·네온·만화풍 금지). 차별화는 스킨이 아니라 *표준화 프레임워크 + 정보 구조*(기회 맵·NPC 액션 큐)에서.
- **상태 색(의미 고정)**: 적격=초록 계열 / 조건부·확인 필요=주황 / 잠금·준비=파랑 / 마감 임박=빨강.
- **분야 타입 색(카테고리)**: 사업화=보라, 행사·네트워크=청록, R&D=앰버, 판로·해외=핑크, 시설·공간=회색 등(한눈 분류).
- **카드 상태 시각 규칙**:
  - 적격 = 또렷·강조 테두리(2px) + "적격" 배지 + "바로 신청".
  - 조건부 = 흐림(opacity↓)·점선 테두리 + "확인 필요" + "○○ 입력 시 확정".
  - 잠금 = 회색·자물쇠 아이콘 + "벤처확인 취득 시 해금".
- 타이포: 한국어(Pretendard 권장), 두 가지 굵기(400/500). **유치한 폰트·과한 그림자/네온 금지.**
- 모션: 카운트업, 조건 충족 시 카드가 *잠금 해제되며 또렷해지는* 짧은 연출, "곧 닫힘" 긴급 펄스(절제).

## 5. 화면 명세

### 화면 1 — 첫 화면 (집계 훅 → 입력 → 개인화)
- 상단: 큰 숫자 훅. 예) "지금 열린 지원사업 **1,545건** · 마감 임박 28건". (익명·사실)
- 중앙: 사업자번호 단일 입력 + "내 사업 찾기" 버튼. 부가 설명 최소.
- 입력 직후(비저장 티저): 개인화 카운트업 — "당신 사업 기준 **적격 63 · 조건부 88** · 추정 최대 ₩X". (저장·정밀조회는 로그인 뒤)

### 화면 2 — 기회 맵 (퀘스트 보드)
- 카드 그리드. 각 카드 = 지원사업(퀘스트). 상태(적격/조건부/잠금) 시각 규칙 적용.
- 카드 앞면: [분야 타입 칩] [상태 배지] / 사업명 / **조건 충족 진행(예 3/3 ✅)** + 조건별 행(지역·업력 등) / **"받을 수 있는 것" 혜택 아이템 행**(자금 ₩·컨설팅·전시·공간 등, certainty 표기, WoW 보상 아이템처럼) / D-day 마감 / CTA. (혜택 옆 자부담·정산 등 의무도 균형 표기)
- 상단 토글: **2D 로드맵 뷰** — 가로축=시간, 세로 구간 "지금 적격 / 곧 적격(시간 충족) / 준비하면 적격(자격 취득)". 잠금 카드는 미래 노드.
- 파워유저용 "조밀한 목록 뷰" 토글도 제공(카드가 기본).

### 화면 3 — 다음 할 일 (NPC 액션 큐) ★가장 중요
- 제목 "다음 할 일 · 레버리지·긴급순". 나침반/가이드 아이콘.
- 행동 리스트(정렬됨), 각 항목 = [아이콘] 무엇 · 왜(unlock 건수·금액) · **어디서(발급처/접수 딥링크)** · 쉬움(소요 태그) · CTA.
- 정렬: (해금 가치 × 긴급) ÷ 쉬움. 1분 고레버리지 행동이 위로.
- 하단 한 줄: "모든 안내는 공고 원문·자격 판정에서 계산됩니다."(신뢰 표시)

### 화면 4 — 신청 준비 시트
- 한 화면에 *왜 적격·얼마·언제까지·어떻게·뭐가 필요*.
- rule_trace 체크리스트: 각 조건 ✅충족 / ⚠확인필요 / ❌미해당 + 원문 근거.
- 필요 서류 리스트: "이미 보유 ✓ vs 준비 필요"(발급처 표시), 신청 딥링크.

### 화면 5(보조) — 성장 트랙
- 투자단계 가로 트랙: 예비창업 → Seed(현재) → Series A → B → C(잠금).
- 획득 업적 배지(법인전환·벤처확인·첫 선정 등) + "다음 해금되는 퀘스트" 미리보기.

## 6. 핵심 컴포넌트
지원사업 카드(상태 3종) · 액션 아이템 행 · 성장 트랙 스텝퍼 · rule_trace 체크 칩 · 카운트업 숫자 · 상태/타입 배지.

## 7. 실제 샘플 데이터 (이걸로 채워줘)
**사용자 회사**: (가칭)테크스타트 · 경기 소재 · 업력 2년 2개월 · 대표 35세 · 업종 ICT/SW(미입력 가정) · 투자단계 Seed.

**카드 3종**
- 적격: "2026년 스타트업 테크 브릿지 참여기업 모집" · 분야 행사·네트워크 · 조건 3/3(지역 경기 ✅ / 업력 7년 이내·예비 허용 ✅ / 대표 35세 ✅) · D-3.
- 조건부: "디지털 창업기업 사업화 지원" · 분야 사업화 · 2/3(지역 전국 ✅ / 업력 ✅ / 업종 ⚠확인) · "업종 입력 시 확정 — 최대 5천만 원" · D-12.
- 잠금: "벤처기업 전용 기술개발 지원" · 분야 R&D · 2/3(지역 ✅ / 업력 ✅ / 벤처확인 ❌미보유) · "벤처확인 취득 시 5건 해금" · D-20.

**액션 큐(정렬됨)**
1. 업종 입력하기 [1분] — 지원사업 3건 즉시 활성화 · 최대 1.2억 → 입력
2. 스타트업 테크 브릿지 신청 [마감 D-3] — 이미 적격·서류 2종 준비됨 · 접수: 경기 데이터플랫폼 → 신청
3. 벤처확인 취득하기 [+5건 해금] — 발급: 벤처확인종합관리시스템 → 발급처
4. 사업자등록증 업로드 [30초] — 프로필 자동완성·정확도↑ → 업로드

**성장 트랙**: 예비창업 ✓ · Seed(현재) · Series A/B/C 잠금. 업적: 법인 전환 ✓ · 벤처확인 미 · 첫 지원사업 선정 ✓.

## 8. 기술 가이드 (선택)
- Next.js + Tailwind + shadcn/ui 호환 컴포넌트. 한국어. 라이트+다크 모드. 모바일 우선, 반응형.
- 인터랙션: 화면 1→2→3 흐름 연결, 카운트업·카드 활성화 모션, 액션 클릭 시 토스트.

## 9. Do / Don't
| Do | Don't |
|---|---|
| 깔끔한 토스급 톤, 명료·신뢰 우선 | 만화풍·네온·과한 그림자·양피지 텍스처 |
| 게임감은 정보 구조·상태로(기회 맵·액션 큐) | 무거운 게임 스킨·판타지 본문 폰트 |
| 적격만 "빛남", 조건부는 흐림 | 모든 카드 똑같이 화려(거짓확신) |
| 금액에 "최대 ₩X (N건 기준)" 근거 | 단일 과장 숫자 |
| "확인 필요/원문 확인" 정직 표기 | 모르는 걸 단정 |
| 행동을 레버리지·긴급순 정렬 | 무작위/알파벳 목록 |
| 가짜 포인트 대신 진짜 해금·지원금 | 전체 사용자 랭킹·가짜 XP |

## 10. 산출 요청
위 4개 핵심 화면(+성장 트랙)을 **하나의 인터랙티브 프로토타입**으로. 화면 1→2→3 흐름이 클릭으로 이어지게. 샘플 데이터(§7) 그대로 사용. 신뢰 가드레일(§3·§9) 준수.
