## ▶ 붙여넣는 프롬프트 (요약본)

> 한국 정부지원사업 매칭 SaaS "창업노트"의 신규 기능 **"창업노트 매니저"**의 데스크톱 하이파이 워크스페이스 1장(1440px)을 만들어줘.
>
> 이건 **지원사업 신청 *준비를 끝내는* 캔버스형 워룸**이다. 자동제출이 아니라, 사업자 정보·지원 가능한 공고·필요 서류·업로드 파일·Q&A 답변·AI 초안·코칭 코멘트가 **노드로 연결된 작업 공간**에서 "서류 세트를 완성"한다.
>
> **핵심은 "구조화된 스켈레톤 + 조작 가능한 자료"** — 자유 무한 캔버스가 아니다:
> - 공고↔필요서류 같은 **계산된 연결은 시스템이 자동으로 그린다**(사용자가 드래그하지 않음).
> - 사용자는 **내 자료(파일·답변·초안)를 노드 포트에서 드래그해 서류 슬롯에 연결하고, 공고 간 재사용 배정**한다. 노드를 인라인 편집하고, 줌아웃으로 전체를 조망해 판단한다.
> - 레이아웃은 **스윔레인(좌→우: 내 사업자 → 지원사업 → 필요 서류 → 준비물·검토)으로 자동 정렬**되고, 회사가 좌측 단일 루트다.
>
> **톤**: 밝은 work-zone, refined-minimal, 블루 중심+민트 보조, 반경 14–16px, 은은한 점 그리드. 다크 AI툴 클리셰·과한 보라 그라데이션·마케팅 히어로·무질서한 화이트보드 금지. 한국어 UI.
>
> 아래 상세 명세·샘플 데이터·금지사항을 따르고, "이 화면에서 신청 준비를 실제로 끝낼 수 있겠다"가 느껴지게.

## 1. 제품 맥락
- 창업노트 = 사업자번호로 정부지원사업을 매칭. 매니저 = 그 *다음*, **신청 준비를 완료**하는 작업형 SaaS.
- 자동제출/정부사이트 대리조작 ❌ — 서류 세트 완성 + 딥링크·마감 요약까지.

## 2. 핵심 컨셉 — 구조화된 노드 워룸
- 검색 리스트가 아니라, 노드가 연결된 **준비 상태 맵**. *토폴로지보다 준비 상태가 먼저 읽혀야.*
- 킬러 가치: **"사업자등록증 1개 → 3개 공고에 재사용"** 같은 재사용 웹이 한눈에. 표로는 안 나오는 것.

## 3. 톤 & 비주얼
- 밝은 work-zone(흰/연회색), 은은한 점 그리드. 카드/패널 14–16px 반경, 조용하고 정보 밀도 있게.
- **블루 중심 + 민트 보조 강조.** 상태색: 완료=green, 누락/확인필요=amber, 잠금/준비=blue, 일반 연결=muted.
- refined-minimal 업무형. Toss처럼 쉽고 부드럽지만 마케팅/장식/게임 UI 없이.
- 금지: 다크 AI툴 클리셰, 과한 보라 그라데이션, 장식 blob, 무한 자유 캔버스 느낌, 히어로 랜딩.

## 4. 화면 구조 (풀스크린 1440px)
**① 상단 바**: "창업노트 매니저" / 사업자명·번호 요약 / 상태 요약 칩(지원 가능 5·준비 중 3·누락 서류 4·마감 임박 2) / 우측 CTA(서류 세트 다운로드·코치 초대·공유). 이 요약은 **항상 노출**(판단 앵커).
**② 좌측 패널 — "이번 주 할 일"**: 액션 큐(예: 사업자등록증 최신본 업로드 / 사업 개요 답변 보강 / D-3 공고 신청서 초안 검토). 각 항목에 *영향받는 지원사업 수 + 마감 위험*. 하단 필터(전체·적격·확인 필요·준비 필요·마감 임박).
**③ 중앙 캔버스 — 스윔레인 자동 그래프**(아래 §5·§6).
**④ 우측 인스펙터**: 선택 노드 상세. 예 "2026 초기창업패키지": 상태 준비 중 / 마감 D-6 / 서류 5/7 완료 / 연결 답변 4 / AI 초안 상태 / 버튼[초안 열기·누락 서류 보기·이 공고 ZIP 만들기].

## 5. 캔버스 — 노드 타입 & 엣지
**스윔레인(좌→우 열)**: 내 사업자 → 지원사업 → 필요 서류(표준) → 준비물·검토. 노드는 레인에 정렬.
**노드 타입(시각 구분)**:
- 사업자 프로필(좌측 단일 루트): 업력·지역·업종.
- 지원사업: 사업명·기관·D-day·적합도·상태(준비중/확인필요/준비필요) + "서류 5/7" 배지.
- 필요 서류(표준 타입): 사업자등록증·재무제표·4대보험 명부·사업계획서 등. "공통·N개 공고" 배지 + 보유✓/누락⚠.
- 업로드 파일 / Q&A 답변 / AI 초안 / 코칭 코멘트.
**엣지 의미(상태색)**:
- **계산된 연결(자동)**: 공고↔필요서류 = 시스템이 그림. 공통 서류 1개 → 여러 공고로 **fan(재사용 웹)**.
- **사용자 연결(드래그)**: 파일↔서류 슬롯, 답변↔초안. 매칭되면 선 활성+체크.
- 색: 완료 green / 누락 amber / 일반 muted blue.

## 6. 인터랙션 (직접 조작 + 구조)
- **드래그 연결**: 노드 포트(핸들)에서 베지어로 — *내 자료 → 서류 슬롯·재사용 배정*. (계산된 요구 연결은 드래그 대상 아님)
- **인라인 편집**: 노드에서 답변 보강·초안 수정, 또는 우측 인스펙터에서 넓게.
- **줌아웃 오버뷰 + 포커스**: 전체 준비 상태 조망(판단) / 공고 클릭 시 서브그래프만 강조, 나머지 흐림.
- **자동 레이아웃 기본 + "정렬" 버튼**(흐트러지면 재배치). 자유 팬·줌 허용.

## 7. 다운로드 경험 (CTA "서류 세트 다운로드" 누르면)
통합 ZIP / 공고별 ZIP / 공통 서류 폴더 / 공고별 AI 초안 / 누락 서류 목록 / 신청 링크·마감 요약 PDF.

## 11. 산출 요청
